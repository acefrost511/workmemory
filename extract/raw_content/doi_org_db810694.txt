Abstract
--------

Academic integrity aims to ensure that students’ achievements are the result of their own efforts and knowledge. However, academic dishonesty continues to evolve, becoming increasingly difficult to detect and mitigate—especially with the rise of sophisticated, AI-driven tools. The lack of a universal knowledge base and standardized guidelines for addressing emerging forms of misconduct further complicates enforcement, leaving educators with limited resources to effectively identify and respond to violations. This research addresses these challenges by proposing CheatGuard, a centralized online anti-cheating platform with incident reporting capabilities. Inspired by MITRE’s ATT&CK framework used in cybersecurity, CheatGuard catalogs a wide range of cheating techniques reported by institutions—from plagiarism to unauthorized collaboration during exams. For each assessment category, it also provides corresponding detection and prevention methods. CheatGuard’s knowledge base is continuously updated using an AI agent that gathers data from online sources. Tactics, techniques, and procedures related to academic dishonesty are structured similarly to those in MITRE ATT&CK, helping institutions systematically select appropriate countermeasures. This paper presents the conceptual architecture of CheatGuard, explains how it can be accessed and used by educators, and explores its potential to combat academic misconduct in the age of Artificial Intelligence.

Appendix A. CheatGuard AI agent
-------------------------------

### Appendix A. CheatGuard AI agent

CheatGuard-AI, is a Python-based system designed to automate the process of retrieving, filtering, and extracting relevant research articles from the web. The implementation is available in two different versions: one using GPT-4o via OpenAI API and another using Llama 3 via Ollama for local inference. Both implementations follow the same core workflow, where the agent first searches for relevant research papers using Google Search API, then extracts content from the retrieved web pages, and finally processes the extracted content using a LLM to filter and structure the information.

### 1.1 Appendix A.1. GPT-4o based

The first implementation of the AI agent leverages GPT-4o via OpenAI API to process and extract relevant research information. In this approach, the agent begins by searching Google for articles based on a user-defined query. Once the relevant URLs are identified, the system downloads and cleans the content by removing unnecessary elements such as headers, footers, advertisements, and navigation menus. The cleaned text is then sent to GPT-4o, which processes the content to extract the most relevant information, filtering out any redundant or irrelevant sections. Finally, the extracted data, including the title, description, authors, and publication year, is structured and returned in JSON format.

Using GPT-4o offers several advantages. Since the model is hosted on OpenAI’s cloud servers, it does not require local computational resources, making it highly efficient in terms of memory usage. The API-based approach also provides fast response times, allowing for near-instantaneous data extraction and filtering. However, a key limitation of this approach is cost, as OpenAI charges users based on API usage. Additionally, the reliance on an internet connection makes this solution unsuitable for offline environments. Despite these drawbacks, GPT-4o provides an efficient, scalable, and high-performance solution for research paper extraction.

### 1.2 Appendix A.2. Ollama based

The second implementation replaces OpenAI’s API-based approach with Ollama, an open-source tool for running local LLM inference. Instead of querying an external server, this approach downloads and runs a LLM on the user’s local machine. For this implementation, Llama 3.2 (2GB) was selected due to its relatively small model size and efficiency. The overall workflow remains the same—the system searches for research articles, extracts content from web pages, and then processes the extracted text using the locally hosted Llama 3.2 model.

One of the major benefits of this approach is that it is completely free to use. Since the model runs locally, there are no API costs, making it an attractive option for users who require a cost-efficient solution. Another advantage is that this approach works offline, allowing users to process research data without an internet connection once the model is installed. However, running LLMs locally introduces performance limitations. Unlike GPT-4o, which leverages cloud-based GPU acceleration, local inference with Llama 3.2 relies entirely on CPU and RAM, making it significantly slower. Additionally, larger models like Llama 3.3 (42GB) were too heavy to be used efficiently on standard consumer hardware.

Despite these challenges, Ollama provides a highly flexible and customizable solution. Users can choose between different models based on their hardware capabilities and processing needs, making it ideal for those who prefer a self-hosted AI solution without ongoing costs.

### 1.3 Appendix A.3. Comparison

Both implementations successfully extracted and summarized relevant research papers, as the searching mechanism was independent of the LLM—only the filtering and extraction were handled by AI models. However, there are significant trade-offs between the two approaches, summarized in Table [5](/article/10.1007/s10639-025-13705-3#Tab5) below.

This makes it an ideal choice for users who need real-time research summarization and have the budget for API usage. In contrast, Ollama provides a free, offline alternative, but requires more memory and processing time. The choice between the two solutions ultimately depends on whether the user prioritizes cost efficiency or performance.
