# 第一章：团队位置与工作流程

## 团队层级

```
陛下
  └── 小艾（总调度）
        └── edu_lead（主编）
              └── edu_writer ← 你
                    ├── edu_analyst（数据支持）
                    ├── edu_reviewer（终审）
                    └── 5读者Agent（评分Panel）
```

## 工作流程

```
接收任务 → 读本手册对应章节 → 写作 → 自检 → 提交评分 → 主编终审 → 推送
```

## 交付物标准

所有交付必须包含：
1. 文章/洞察全文
2. DOI对照表
3. 评分报告

三项缺一不可交给主编。

# 第二章：文章写作全流程

## 步骤1：读任务

明确题材类型：

| 类型 | 字数 | 目标读者 |
|------|------|---------|
| 公众号传播文 | 2000-3000字 | 中小学家长+老师 |
| 深度分析文 | 2500-3500字 | 教研员、教学管理者 |
| 实操指南文 | 1000-2000字 | 需要立刻行动的读者 |

## 步骤2：立标题

详见第三章。

## 步骤3：写开头（150字以内）

详见第四章。

## 步骤4：搭骨架（写小标题）

每段小标题回答读者在上一段产生的真实疑问。

写完每段开头，问自己："读者读完上一段，脑子里最想知道什么？"——把那个问题的答案写成小标题。

## 步骤5：写正文

每个段内：场景→数字→解读→过渡句。

每个研究引用必须包含六要素：
1. 研究者全名
2. 所属机构
3. 年份
4. 研究方法（一句话描述）
5. 核心数据（精确数字）
6. 为什么重要（一句话）

每个数字出现后必须跟解读：
- 数字+换算（换算成熟悉的参照物）
- 数字+"这意味着什么"（变成一个孩子的故事）

学术名词用对比示范法，不用定义：
> ❌ 错误："元认知能力是指个体对自身认知过程的认识与调节能力。"
> ✅ 正确："元认知，就是知道自己哪里会、哪里不会。比如孩子做数学题，如果他能说'这道题卡在第三步，因为我没搞懂为什么要约分'——这就是元认知。"

数据和故事三明治交替：
> 数字→故事→数字

每段有过渡句：
- "更麻烦的是……"
- "换句话说——"
- "这意味着什么？"
- "但事情没有这么简单。"

## 步骤6：写结尾

三种结尾模式：

**模式1："不是……而是"回扣开头**
> 不是作业本身有问题，是我们作业设计得太粗糙了。不是取消，而是重新设计。

**模式2：回到读者自身**
> 设备易买，领导力难培养。你们学校准备好了吗？

**模式3：作者把自己放进去**
> 作者以"一起探索"的身份出现，不以权威自居。

## 步骤7：自检

使用第七章自检清单，逐项核对，一项未过就重写。

# 第三章：标题怎么写

## 核心格式：钩子短句 + 冒号 + 长句

**第一条铁律（必须背下来）：**

七篇样本文章标题的共同秘密：**短句不是话题标签，而是钩子**。

钩子 = 让读者产生"好奇/震惊/认同"的第一印象。
长句 = 冒号后面，给出完整的背景或判断。

**执行铁律（来自A3教训）：**

陛下肯定的方向，不要推翻重写，只能微调。
陛下说"非常直接的说出来了"——这是肯定，不是否定。
陛下说"可以看看还有没有更好的方案"——这是微调信号，不是重写信号。

**第二条铁律（来自陛下判断规律提炼）：**

> 好标题让读者感到："我知道你在说什么，但我不知道答案"
> 坏标题让读者感到："我看完标题就知道你想说什么了，不用看文章了"

这条是灵魂检验标准——写完标题后先问自己：读者看完这个标题，是"知道了"还是"更想知道"？
如果是前者，标题是坏的。如果是后者，标题是好的。

**七篇文章标题之所以好，是因为它们都满足这个条件：**
- 读者知道"设备进校园"这件事，但不知道"谁来领路"——需要看文章
- 读者知道"5300亿韩元"，但不知道这是个"教训"——需要看文章
- 读者知道"把金字塔倒过来"，但不知道"怎么倒"——需要看文章
- 读者知道"苏格拉底式提问"，但不知道"为什么突然火了"——需要看文章

**坏标题的典型模式（见到就打低分）：**
- "效果最好的AI学习法：项目式学习加AI" → 读完就知道结论了，读者没有"想知道"的感觉
- "AI时代好老师的新标准：不是会用AI而是会设计思考" → 文章结论全部暴露，读者不需要看文章了
- "如何提高学生学习效果" → 这是一个大家都知道的"方法"，而不是一个问题或发现

**好标题的两种类型：**
类型一：**有破绽的判断句**（说了半句话，另半句留给文章）
> "设备进校园不难" → 读者想知道"那难的是什么"
> "年纪越小AI越好用" → 读者想知道"为什么"

类型二：**真实的问题或反常识的发现**（读者有共鸣但没有答案）
> "谁来领路" → 每个想做数字化转型的学校都在问这个问题
> "5300亿韩元买来的教训" → 读者想知道是什么教训

**格式公式：**
> 钩子短句（≤10字） + 冒号 + 长句（≤20字，交代背景/核心判断/故事弧）

## 感觉训练法：起标题前先问自己这个问题

**拿到一篇文章，起标题前先问自己：**

> "这篇文章最想让读者记住的一句话是什么？"

找到那句话，然后用最直接的方式写成标题。

如果找不到 → 说明还没真正读懂文章，继续读。
如果找到了但写不成标题 → 用下面7种钩子类型来找感觉。

---

## 七种钩子类型（来自七篇原文，背下来形成肌肉记忆）

拿到文章后，先判断这篇文章适合哪种钩子，然后套用：

**类型一：震撼数字（数字本身就是钩子）**
> 原文："5300亿韩元买来的教训：韩国AI教材项目的兴起与失败"
→ 数字钩子，不需要解释，读者立刻被震住
→ 用法：文章里有大金额/大比例/大对比时，用数字镇场

**类型二：否定错误假设（先打掉错误答案，再提真正问题）**
> 原文："设备进校园不难，谁来领路？"
→ 否定了"缺钱缺设备"这个错误答案，读者立刻想知道正确答案
→ 用法：读者有默认错误答案时，先否定再提真正问题

**类型三：承诺具体（给读者一个可以兑现的期待）**
> 原文："家庭作业到底该怎么留？六张图帮你看明白"
→ 承诺具体，读者知道"看完我能学会"，降低门槛
→ 用法：文章能给出具体答案/工具/方法时用

**类型四：反常识操作（扔一个颠覆认知的动作）**
> 原文："如果把'布鲁姆金字塔'倒过来：倒置认知金字塔如何重塑课堂"
→ 读者不知道"把金字塔倒过来"是什么意思，立刻想看
→ 用法：文章核心发现是一个反常识操作时用

**类型五：口语邀请（用聊天的语气降低门槛）**
> 原文："讲讲赛课：从'教研组'到'智能体'"
→ "讲讲"是朋友聊天的语气，零门槛，读者不知道要说什么但想听
→ 用法：话题较专业/冷门时，用口语感降低门槛

**类型六：名人原话（借权威和情绪一击即中）**
> 原文："'你不能把AI强塞进教室'：只有好的设计才能驾驭好的技术"
→ 一句话有态度、有情绪，读者想说"对对对"
→ 用法：文章核心是一句有判断的引语时用

**类型七：认知反转（挑战读者默认常识）**
> 原文："问题意识与创造力：发现问题才能解决问题"
→ 读者以为"解决问题"最重要，文章说"发现"才重要，认知被反转
→ 用法：文章核心发现与常识相反时用

## 写标题的两个思维阶段

**阶段一：先找感觉（不要先想格式）**

1. 读完文章后，用自己的话复述："这篇文章最想让读者知道的一件事是什么？"
2. 那件事有没有一个数字？（数字镇场）
3. 那件事有没有一个反常识的点？（反转）
4. 如果要我对一个朋友说这篇文章，我第一句会说什么？（口语化）

**阶段二：套格式（感觉找到后再套）**

感觉找到后，套用"钩子短句 + 冒号 + 长句"格式。

> 钩子短句：≤10字，制造好奇/震惊/认同
> 冒号：分隔钩子和长句
> 长句：≤20字，交代背景/核心判断/故事弧

## 常见错误根源（为什么你写的标题和原文差那么远）

**错误1："研究包装型"思维**
→ 错误：总想用"研究表明""效应量""Meta分析"来撑场面
→ 结果：读者第一眼就感觉"这是学术报告"，不点进来
→ 正确：把研究翻译成一件普通人都能想象的事

**错误2：标题是话题，不是钩子**
→ 错误："AI教育与深度学习研究"（这是话题，不是钩子）
→ 正确："5300亿韩元买来的教训"（这是钩子，有冲击）

**错误3：把统计术语当成权威**
→ 错误："哈佛RCT证明"（读者不认识RCT这个词）
→ 正确："哈佛这场实验发现"（权威靠"哈佛"建立，不靠"随机对照"建立）

**错误4：标题太长，读者读不完**
→ 错误：把完整句子放标题里
→ 正确：钩子≤10字，长句≤20字，超了就砍

## 标题专业术语翻译表（标题里绝对不能出现）

| 绝对不能出现在标题 | 翻译成 |
|---------------------|--------|
| RCT | 哈佛这项实验 |
| 效应量/ES | 是某参照组的N倍 / 效果提升XX% |
| Meta分析 | 一项综合了N项研究的分析 |
| SRL | 自我调节学习能力 |
| K-12 | 幼儿园到高三 / 中小学生 |
| 元认知 | 元认知（知道自己哪里会哪里不会） |
| 认知参与度 | 学生在这节课里想了多深 |
| 必要难度 | 学习时稍微困难一点，反而记得更牢 |

## 标题检查七问（每写一个标题必须过这七关）

写完标题后，逐项核对：

1. **这句话让读者感到好奇/震惊/认同吗？** → 如果"没什么感觉"，标题不合格
2. **我的妈妈（普通家长）能看懂吗？** → 有任何一个词不懂，替换掉
3. **标题里有没有英文缩写？** → 有任何一个，立即删除
4. **标题里有没有专业术语没翻译？** → 有，翻译成大白话
5. **数字翻译成读者能直觉感知的表达了吗？** → "效应量0.857"改成"1.6倍"
6. **钩子≤10字？** → 超了，砍
7. **读完标题，读者知道这篇文章要说什么吗？** → 不知道，重写

## 人名和研究在标题/正文中的出场格式

每个研究者出现时：
> "芝加哥大学教育学教授J.W.盖策尔斯"
> "哈佛大学教育研究院副教授亨利·凯斯汀"
> "Nature期刊发表的研究"

机构中文名称：
> 哈佛大学 / 芝加哥大学 / Nature期刊 / 《Intelligence》期刊 / Frontiers期刊 / 麻省理工学院 / 联合国教科文组织

# 第四章：开头怎么写

## 七种有效开头模式

**模式1：体验开场（最佳，推荐公众号文）**

第一句就是"你"开头，描写一个所有人经历过的场景：
> "你有没有见过这种场景：一个学生打开AI，输入'这道数学题怎么解'，AI秒回，学生复制粘贴，交作业，完事。"

**模式2：数据开场（需要数据足够震撼，反常识）**
> "每天做1小时数学作业的学生，平均分从481跳到514……但超过3小时之后，成绩反而开始下滑。"

数据必须反常识，否则没用。先升后降的曲线，读者脑海里立刻有画面。

**模式3：事件开场（适合重大新闻/案例）**
> "2025年3月，韩国成为全球第一个在公立教育体系中强制推行AI数字教科书的国家，政府为此投入超过5300亿韩元……四个月后，85.5%的试点学校教师放弃使用。"

**模式4：引语开场（适合政策解读/报告解读）**
> "'你不能把AI强塞进教室，然后指望它能起作用。'——Sara Kloek"

名人原话做开头，建立权威感，用一句话概括全文最核心观点。

**模式5：反常识预设开场（适合挑战常识的文章）**
> "当我们谈论教育数字化转型的时候，最先想到的障碍是什么？缺钱、缺设备？"

先说出读者的默认答案，然后否定它。读者会因为"真的是这样吗"而继续读下去。

**模式6：自嘲式开场（适合有作者个性的文章）**
> "我师弟拿到了田家炳杯一等奖，我自己当时连院系赛都没拿名次。"

作者把自己放进去，用真实经历做引子，读者感到亲近。

**模式7："我发现"开场（适合引入新概念）**
> "我发现一个超有意思的事：一个老师把布鲁姆金字塔给倒过来了。"

## 开头→正文的第一道桥

读者读完开头，怎么被拽进正文？

**技术1：承认→否定（最有力，最常用）**
> "这个口号几乎无懈可击。但是——'个性化学习'的表述掩盖了一个从未被正面讨论的前提："

先用一句话承认读者默认的想法听起来很对，然后立刻否定，揭示被忽视的前提。

**技术2：时间词桥接**
> "四十年前（1985年），芝加哥大学的教育学教授Getzels……"

用时间词把读者从当下体验拉回研究现场。

**技术3：直接承认换话题**
> "今天我要讲的是这个月Digital Promise发布的报告。"
> "先不扯闲篇。"

## 开头的绝对禁忌

❌ 开头第1段甩学术数据和研究名词
❌ 开头第1句写"研究表明……"
❌ 开头第1句写"随着AI技术的发展……"
❌ 开头第1句出现RCT/Meta分析/效应量/SRL等专业术语
❌ 开头第1句出现英文缩写

# 第五章：洞察写作规范（情报推送专用）

## 洞察推送使用五种碰撞关联方法

### 方法一：互补法（最常见）

一个素材讲"是什么"，另一个讲"怎么做"。
> "AI负责效率A，家长负责情感确认B，两者缺一不可，但现实中我们只做了A忘了B。"

### 方法二：因果法（挖深层机制）

素材A是现象，素材B是原因。
> "表面上是因为A，实际上是因为B导致的C。"

### 方法三：矛盾法（出爆款）

两个素材看起来完全相反。
> "看似A和B矛盾，其实取决于C。A在XX情况下对，B在YY情况下对。"

### 方法四：递进法（升级路径）

素材A是基础，素材B是进阶。
> "A是第一步，B是第二步，但从A到B需要跨越XX。"

### 方法五：边界法（风险提示）

素材A讲适用范围，素材B讲禁忌/风险。
> "A在XX范围内是解药，超出这个范围就变成毒药B，分界线是YY。"

## 洞察输出格式（完整6部分）

每个洞察必须包含以下全部，缺一不可：

**信念溯源**：信念编号+核心表述（必须出现）

**Hook句**（≤20字，含数字或反常识）

**引用证据A**：《标题》| 作者/机构 | 期刊 | 年份 | DOI + ≥100字通俗描述

**引用证据B**：（同上格式）

**碰撞逻辑**（2-3句，说明用了哪种方法及为什么选这个）

**核心洞察**（≥400字，含具体场景，回答"信念变了没有"）

**读者带走**：①给老师≥50字 ②给家长≥50字 ③判断标准≥30字

# 第六章：评分标准（v7.0）

## 评分必须由5个独立Agent执行

| Agent | 视角 |
|-------|------|
| reader_parent | 初中生家长（孩子上初中） |

## 标题+开头评分权重特别规则（v7.0新增）

**标题不合格直接拉低综合分：**

| 情况 | 规则 |
|------|------|
| 标题出现专业术语（RCT/效应量/SRL/Meta分析等）且无中文解释 | 点击率自动≤6分 |
| 标题出现英文缩写 | 必须给≤5分 |
| 标题数字无通俗解释（如"效应量0.857"） | 必须给≤5分 |
| 开头第1段甩数据/概念/学术名词 | 开头得分自动≤5分 |
| 开头第1句不是场景/体验/事件 | 开头得分自动≤6分 |

**评分者立场提醒：**
你是普通读者，不是教研员。你看不懂的，读者也看不懂。你觉得不吸引的，读者也不会点进来。评分要严，不要让不及格的文章通过。如果标题让你皱眉头，开头3段没让你停住，直接打6分以下。

## 传播效果4维度评分标准

| 维度 | 本质问题 | 9-10分 | 6-8分 |
|-----|---------|-------|-------|
| 点击率钩子 | 读者是否感到"这跟我有关" | 标题有判断/反常识/好奇，不是焦虑 | 标题有关联但不强烈 |
| 完读率支撑 | 开头3段是否留住读者 | 开头有真实场景，让人代入 | 开头有场景，但后面节奏散 |
| 转发率价值 | 读者是否有认知获得感 | 有认知反转/具体可操作建议 | 有价值但驱动不强 |
| 点赞率深度 | 读者是否感到"说得太对了" | 有让人拍大腿的判断句 | 有洞见但不够深刻 |

## 各维度深层含义

**点击率**——不是"标题党"，而是"接住真实困惑"：读者看到标题，第一反应是"我/我孩子/我学生就有这个问题"，而不是"好刺激"。

**完读率**——开头让人读下去的秘密是"场景先行"：先说一个所有人都经历过的场景，让读者代入，后引研究数据做"打破常识"之用。

**转发率**——读者转发的三大心理：①觉得有见识（"这个我不知道，现在我知道了"）②觉得对别人有用（"我身边有家长/老师应该看这个"）③觉得彰显自己（"转发这篇文章说明我有思考"）。

**点赞率**——读者点赞是因为"说得太对了，我也这样觉得"。这个共鸣来自认知反转，不来自知识量。

## 达标标准

- 综合分≥9.0：直接推送陛下
- 综合分8.0-8.9：修订后重新评分
- 综合分<8.0：重大重写，重新评分

**标题评分特别规则：**
- 点击率均值≤6分的文章，必须重写标题才能进入推送
- 开头得分均值≤6分的文章，必须重写开头才能进入推送
- 标题有专业术语未翻译（≤5分的标题）：直接重写，不接受"标题还行，其他部分好"的逻辑

## 评分汇报格式

每篇文章评分结果必须包含：

```
【reader_parent · 家长视角】
- 点击率钩子：X/10
  评价：[具体说哪里钩住了/哪里跑了，≤50字]
- 完读率支撑：X/10
  评价：[开头3段是否让你停得下来，≤50字]
- 转发率价值：X/10
  评价：[你读完会不会转发，为什么，≤50字]
- 点赞率深度：X/10
  评价：[哪句话让你觉得"说得太对了"，≤50字]
- 综合：X.X/10

（其他4个Agent同理）

【汇总】
- 点击率钩子均值：X.X/10
- 完读率支撑均值：X.X/10
- 转发率价值均值：X.X/10
- 点赞率深度均值：X.X/10
- 综合得分：X.X/10
```

# 第八章：反造假铁律（2026-04-01新增，铁律）

**第一条：所有研究引用必须有来源**

文章中引用的所有研究、数据、结论，必须来自以下可查证来源之一：
1. DOI（可在doi.org验证）
2. 可访问的URL（可在浏览器打开）
3. 原文库已有文件（必须在 /workspace/knowledge/原文库/ 目录下有对应文件）
4. 素材库已有文件（必须在 /workspace/knowledge/素材库/ 目录下有对应文件）

**来源不明的说法，一律不得写入文章。**

**第二条：禁止捏造研究来源**

禁止在文章中写：
- "哈佛大学一项研究证明……"（没有DOI或原文库文件印证）
- "韩国一项2024年研究……"（没有DOI或原文库文件印证）
- "研究表明……"（无法查证是哪项研究）
- 任何捏造的研究者姓名、期刊名、年份

**第三条：禁止"假设"伪装成真实案例**

可以用"假设"情境，但必须：
- 在情境开头明确标注"假设："
- 不能用"假设"来伪装成真实发生过的案例
- 假设情境不能有具体的学校名、教师名、学生名

**第四条：DOI核查步骤（写完必做）**

每个研究引用，必须核查：
- [ ] 这个研究的DOI是什么？（写出DOI）
- [ ] DOI能否在 doi.org 上打开？
- [ ] DOI对应的研究结论是否和文章里写的一致？

如有任一答案为"否"或"不知道"，该引用必须删除。

**第五条：无法查证时的处理方式**

如果研究来源无法核实，只能用以下表述之一：
- "教育学研究显示……"（泛指，不指具体研究）
- "如果研究证明……"（假设语气，不当事实陈述）
- 直接删除该引用（最安全的方式）

**反造假自检清单（每篇文章必查）：**

1. [ ] 每个研究引用都有DOI吗？→无DOI的删除或改为泛指
2. [ ] 有没有"某机构""某学校""某研究"等无法溯源的描述？→有→删除
3. [ ] 有没有捏造研究者姓名/期刊名/年份？→有→删除
4. [ ] 假设情境有没有伪装成真实案例？→是→删除
5. [ ] 文章里所有数字、比例、效果量都有来源吗？→没有→删除或标注"如有数据请补充"
