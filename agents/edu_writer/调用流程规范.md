## 场景一：深度文章写作任务

**触发条件**：陛下下达深度文章写作任务

**流程**：

```
陛下指令
    ↓
小艾（总调度）→ 判断为"文章写作"类型
    ↓
edu_lead（主编）→ 接收任务，拆解目标
    ↓
edu_writer → 调用【文章写作手册】
    → 完成初稿
    → 自检清单20项全过
    ↓
edu_lead → 并行spawn 5个读者Agent
    ↓
5个读者Agent → 各读初稿全文
    → 调用【评估打分手册】评分
    → 各输出评分报告（格式见评估打分手册）
    ↓
edu_lead → 汇总5份评分
    → 计算均值
    ↓
edu_reviewer → 终审
    → 虚构数字检查
    → 反造假检查
    → 格式完整性检查
    ↓
综合分≥9.0 → edu_lead推送陛下（中版）
综合分8.0-8.9 → edu_writer修订→重评
综合分<8.0 → edu_writer重写→重评
```

**Agent调用方式（唯一正确方式）：**

```
# 调用持久Agent
sessions_spawn(
  task="[具体任务描述]",
  runtime="acp",
  agentId="edu_writer",  # 持久Agent的agentId
  runTimeoutSeconds=XXX,
  mode="run"
)

# 禁止用runtime="subagent"调用持久Agent（无记忆无SOUL，造假风险高）
```

## 场景二：每日洞察推送任务

**触发条件**：情报官（info_officer）完成当日情报收集，或陛下手动下达洞察任务

**流程**：

```
情报官完成当日收集 → 入库文件通知
    ↓
insight_officer（洞察官）→ 接收任务
    ↓
insight_officer → 读取今日入库全部文献
    → 读取信念抽屉文件
    → 筛选碰撞素材组合
    → 生成3条洞察候选
    ↓
insight_officer → 并行spawn 5个读者Agent
    ↓
5个读者Agent → 各读洞察全文
    → 调用【评估打分手册】评分
    → 各输出评分报告
    ↓
insight_officer → 汇总5份评分
    → 计算均值
    ↓
insight_officer → 反哺信念抽屉
    → 达标洞察追加写入对应抽屉的「碰撞产出品区」
    ↓
综合分≥9.0 → insight_officer推送陛下（中版）
综合分8.0-8.9 → insight_officer修订→重评
综合分<8.0 → insight_officer重写→循环（最多3轮）
```

## 场景三：情报收集任务

**触发条件**：每日05:00定时触发，或陛下手动下达情报任务

**流程**：

```
cron每日05:00 → info_officer自动启动
    ↓
info_officer → 按intel_01~12分四批并行搜索
    ↓
intel_01~03 → 搜索英文期刊/顶会
intel_04~06 → 搜索中文期刊
intel_07~09 → 搜索产品/报告
intel_10~12 → 搜索政策/行业新闻
    ↓
info_officer → 汇总所有入库文件
    ↓
intel_reviewer_en + intel_reviewer_cn + intel_reviewer_intl
→ 并行审核（DOI验证+质量过滤）
    ↓
info_officer → 推送入库报告给陛下
    → 同时通知insight_officer有新情报
```

## 调用禁令（铁律）

1. ❌ 禁止用 `runtime="subagent"` 调用持久Agent
2. ❌ 禁止在task里写角色设定（角色在Agent的SOUL.md里）
3. ❌ 禁止跳过edu_lead直接spawn edu_writer
4. ❌ 禁止跳过insight_officer直接spawn intel_agents
5. ❌ 禁止用中文标签做agentId
6. ❌ 禁止Spawn后不汇报、不监控、不追踪结果
