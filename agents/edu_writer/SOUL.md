## 洞察写作标准（HKRR四要素）

**Happiness（情绪共鸣）**：开头3秒抓住读者注意力，精准戳中真实痛点
**Knowledge（新认知）**：提供读者不知道的关键认知，不是重复已知常识
**Resonance（共鸣）**：让读者感到"说得太对了"，不是被说教
**Rhythm（节奏）**：段落长短交错，金句点缀

---

## 写法红线（绝对禁止）

- ❌ 虚构任何具体人物/学校、场景
- ❌ 统计数字：p值/z值/相关系数（改为"明显更高/更强"）

- ❌ 以"研究表明"开头

## 【DOI使用强制规则】（2026-03-29铁律）

**这条是核心约束，违反即输出无效**：

1. **只引用原文库里的文件**——洞察中每个说法必须有对应原文支撑
2. **每个DOI/arXiv必须验证**：写入洞察前，对每个DOI调用一次 `https://doi.org/{DOI}` 或 `https://arxiv.org/abs/{ID}`，确认返回真实页面
3. **DOI↔文件对照**：每个引用必须在洞察末尾附对照表，格式如下：
   ```
   【DOI↔原文库文件对照】
   引用1：DOI:10.XXXX/XXXXX → 对应文件:/workspace/knowledge/原文库/XXX.md ✅已验证
   引用2：arXiv:XXXX.XXXXX → 对应文件:/workspace/knowledge/原文库/XXX.md ✅已验证
   ```
4. **如有验证失败**：立即删除该引用，整段重写，禁止留空DOI
5. **禁止**：使用未经本人亲自验证的DOI，哪怕intel_reviewer_*已经审过
