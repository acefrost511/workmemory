## 用户意图识别与路由机制

### 路由判断逻辑
- **教师**：教学设计、备课、课程规划、讲义编写、导学案设计 → 备课业务线
- **学生**：错题讲解、题目不会做、知识点不理解、解题思路求助 → 答疑业务线
- **学生**：学习方法、学习动力、如何自主学习、思维训练 → 启发式学习业务线
- **教师/管理者**：查看学生成长图谱、班级学情分析 → 学情数据管理模块
- **意图不明确**：通过一轮追问确认用户身份和需求后再路由

## 四条业务线

### 业务线一：AI备课（面向教师）
- **终审角色**：备课总设计师（beike_designer）
- **协作流程**：
  1. 并行：分发给四位专家（理论/实践/跨学科/AI）各自独立分析
  2. 汇聚：四份报告汇聚至备课总设计师
  3. 终审整合：备课总设计师综合决策，输出完整教学设计方案
  4. 交付与反馈：提交给教师，接收修改意见

### 业务线二：AI答疑（面向学生）
- **终审角色**：答疑导师（dayi_mentor）
- **协作流程**：
  1. 第零步：学情数据管理模块检索学生历史画像
  2. 并行：四位专家按答疑场景各自独立分析
  3. 汇聚：四份报告+学生画像汇聚至答疑导师
  4. 终审输出：答疑导师面向学生输出引导式讲解
  5. 实时迭代：根据学生回应调整讲解策略
  6. 数据记录：交互结束后提交关键数据至学情数据模块

### 业务线三：AI启发式学习（面向学生）
- **终审角色**：学习力教练（xuexili_coach）
- **协作流程**：
  1. 第零步：学情数据管理模块检索学生历史画像
  2. 并行：四位专家按启发式学习场景各自独立分析
  3. 汇聚：四份报告汇聚至学习力教练
  4. 终审输出：学习力教练面向学生输出启发式对话
  5. 实时迭代：根据学生回应调整对话策略
  6. 数据记录：交互结束后提交关键数据至学情数据模块

### 业务线四：成长图谱报告（面向教师/管理者）
- **执行角色**：学情数据管理模块（xueqing_data）
- **流程**：直接输出个人成长图谱报告或班级成长图谱报告
- **补充分析**：如有需要，可指派理论教学专家和实践教学专家补充分析
