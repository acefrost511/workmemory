# 情报官团队·事件驱动流水线
> 版本：v2.0 | 日期：2026-03-28 | 陛下确认版

## 核心原则：事件驱动，不是时间驱动

- ❌ 旧逻辑：固定时间5点扫描→固定6点归档→固定17:30洞察
- ✅ 新逻辑：搜索完成→审核通过→写入原文库→触发素材库→触发归档→17:30推送

## 完整事件驱动流程

```
事件A：情报官开始扫描
  ↓
情报官并发运行12个搜索Agent + 3个审核Agent
  ↓
事件B（每发生1次）：搜索Agent找到1篇论文
  ↓
审核Agent立即审核这篇论文
  ↓
事件C（通过时）：写入原文库（标注✅已验证）
  ↓ 同步
事件D：情报官立即概括这篇论文→写入素材库
  ↓
事件E：触发IP归档团队（实时归档到信念抽屉）
  ↓
日报归档（每日12:00前检查）：今日共归档了几篇？有没有问题？
  ↓
事件F（17:30）：推送今日洞察给陛下
```

## 事件处理机制

### 事件D→E（素材库新增→触发IP归档）

**触发条件：** 素材库今日新增了1篇或以上的论文

**触发方式：** 情报官在每次写入素材库后，执行：
```
bash /workspace/scripts/trigger_archive.sh
```

trigger_archive.sh 的内容：
```bash
#!/bin/bash
# 功能：检测今日素材库是否有新增，若有则触发IP归档
TODAY=$(date +%Y-%m-%d)
MATERIAL_FILE="/workspace/knowledge/素材库/素材库_${TODAY}.md"

if [ -f "$MATERIAL_FILE" ]; then
    LINES=$(wc -l < "$MATERIAL_FILE")
    LAST_COUNT_FILE="/workspace/.archive_trigger_count"
    LAST_COUNT=$(cat "$LAST_COUNT_FILE" 2>/dev/null || echo "0")
    
    if [ "$LINES" -gt "$LAST_COUNT" ]; then
        echo "$LINES" > "$LAST_COUNT_FILE"
        echo "触发：今日素材库新增内容，触发IP归档 $(date)" >> /workspace/logs/archive_trigger.log
        # 写入触发标记文件
        echo "$(date '+%Y-%m-%d %H:%M:%S') 新增${LINES}行" >> /workspace/.archive_trigger_pending
    fi
fi
```

### IP归档团队（被触发后执行）

**职责：**
1. 读取今日素材库文件
2. 逐篇判断归属，归入13个信念抽屉
3. 检查归档完整性
4. 若发现问题，立即排查解决

**日报检查（每日12:00）：**
```
执行 /workspace/scripts/daily_archive_check.sh
内容：今日共归档了几篇？信念分布合理吗？有无异常？
若发现问题：立即排查，重新执行归档流程
若无问题：静默通过
```

### 事件F（17:30推送洞察）

**前提条件（必须同时满足）：**
- 今日素材库已有内容归档完成
- IP洞察生产完成（Writer→6读者→≥9分）

**若17:30洞察未就绪：**
→ 立即排查原因
→ 延迟推送，直到有洞察才推送
→ 不能为了赶时间推送质量不达标的洞察

## 不再使用固定时间cron的环节

| 旧（时间驱动） | 新（事件驱动） |
|------|------|
| 每天5点扫描 | 情报官持续扫描，有新内容就处理 |
| 每天6点归档 | 素材库每新增1篇就触发归档 |
| 每天17:30洞察 | 归档完成+洞察≥9分→立即推送 |

## 仍然保留的cron（精简版）

| 时间 | 任务 | 性质 |
|------|------|------|
| 每日12:00 | IP日报归档检查 | 兜底检查，确保无遗漏 |
| 每日17:30 | 洞察推送（若有） | 最终交付 |
| 每日05:00 | 情报官启动（启动扫描进程） | 启动信号 |
| 每2小时 | Cron健康检查 | 稳定性保障 |
| 每天03:00 | GitHub全量同步 | 数据安全 |

## 异常处理规则

**若12:00日报检查发现问题：**
1. 记录问题（哪个抽屉？什么异常？）
2. 立即重新执行归档
3. 解决后继续流程
4. 若无法解决 → 向陛下汇报

**若17:30洞察未就绪：**
1. 不推送（不凑数）
2. 向陛下说明原因
3. 继续执行直到有洞察
4. 推送后说明延迟原因

## 情报官协调员职责（新增）

在原有职责上新增：
- 每次写入素材库后 → 执行trigger_archive.sh
- 记录今日累计写入篇数
- 若当日首次写入 → 通知IP归档团队启动
