# 情报官存储配置方案（最终版）
> 版本：v2.0 | 制定：2026-03-27 | 更新：2026-03-27（陛下最终确认版）
> 维护者：情报官

---

## 一、两库架构（最终方案）

```
/workspace/knowledge/
  素材库/              ← 【核心资料库】情报官维护
  原文库/              ← 【原文索引库】情报官维护
```

---

## 二、素材库（核心资料库）

**用途**：所有研究概述 + 陛下的所有素材笔记洞察，全部原文保留

**容量规则**：
- 容量上限：200篇研究概述（超出按发表时间删除最旧15%）
- 陛下内容：无限保留，超1000字压缩提取，不足1000字原文保留

**单篇研究概述格式**（必须全部字段，缺一不可）：
```
【素材N】
类型：研究论文 / 国际报告 / 政策文件

【标题】
英文标题：[原文]
中文标题：[翻译]

【基本信息】
作者：[姓名]
机构：[机构名称]
期刊名称：[期刊/来源名称]
发表时间：[YYYY-MM]
DOI：[10.XXXX/XXXXX]（无DOI则填"无"）
原文索引：[指向原文库的文件路径或DOI链接]

【研究背景】
（≥400字，描述研究背景、问题来源、研究动机）

【研究过程/方法】
（≥200字，描述研究方法、数据来源、样本、工具等）

【研究发现】
（≥400字，描述核心发现、具体数字、统计结果）

【对教育的价值/启示】
（≥200字，描述研究对K12 AI教育的实践意义）

【我的笔记/思考】
（陛下提供的素材原文，或AI洞察结论，≤1000字原文保留，超1000字压缩提取关键要点）

收录时间：[YYYY-MM-DD HH:mm]
来源分类：期刊 / 预印本 / 国际报告 / 政策 / 陛下素材
关键词：[3-5个标签，逗号分隔]
```

**陛下素材处理规则**：
- 陛下发送的原文（笔记/文章/洞察/观点）→ 直接存入【我的笔记/思考】字段
- 不足1000字：全文保留，不压缩
- 超过1000字：压缩为300-500字的精华摘要，保留核心观点
- 陛下亲手写的内容：必须原文保留，不得压缩

**索引机制（核心！每篇研究必备）**：
- 每篇素材必须包含【原文索引】字段
- 格式：`原文库/{年份}/{DOI或文件名}.md`
- 无DOI的研究：用标题拼音+日期作为索引文件名
- 索引用于：点击可跳转定位到原文库对应位置

## 三、原文库（原文索引库）

**用途**：存储每篇论文/报告的完整原始内容，供深度追溯查阅

**容量规则**：
- 容量：50-100篇（基准80篇）
- 预警线：70篇
- 上限：100篇（超限自动删除最早发表的20%）
- 下限：50篇（不触发删除）

**淘汰机制**：
- 超过100篇 → 删除最早发表的20%
- 删除前：将被删研究清单记录到淘汰报告中供陛下查阅
- 删除前向陛下推送淘汰报告

**单篇原文文件格式**：
```
文件名：{DOI号.md} 或 {标题拼音_dd.md}
路径：/workspace/knowledge/原文库/{年份}/{DOI或文件名.md}

内容：
- 英文标题
- 中文标题
- 作者
- 机构
- 期刊名称
- 发表时间
- DOI（已验证可访问）
- 原文链接（URL）
- 完整原文内容（或最接近原文的摘要/全文页面内容）
- 收录时间
- 来源分类
```

---

## 四、维护机制

**每次收录新研究后的检查流程**：
```
1. 写入素材库（research_pool/）
2. 写入原文库（research_raw/）
3. 检查原文库是否≥100篇 → 是则触发淘汰
4. 检查素材库研究概述是否≥200篇 → 是则触发淘汰（删最旧15%）
5. 向陛下推送入库报告（入库数量/淘汰数量/当前总容量）
```

**读取方式（无索引文件，Agent直接读文件夹）**：
- Agent读取 /workspace/knowledge/素材库/ 下的所有.md文件
- 根据【研究背景】【研究发现】【对教育的价值】字段判断是否相关
- 点击【原文索引】跳转到原文库对应文件
- 禁止仅凭标题判断，必须读正文

## 五、容量参数汇总

| 库 | 用途 | 容量上限 | 预警线 | 淘汰触发 | 淘汰量 |
|----|------|---------|--------|---------|--------|
| 素材库 | 研究概述+陛下素材 | 200篇研究概述 | 160篇 | 满200篇 | 删最旧15% |
| 原文库 | 论文完整原文 | 80篇（50-100范围） | 70篇 | 满100篇 | 删最旧20% |

**陛下素材**：无限保留，超1000字压缩，不足1000字原文保留

## 六、两库关系图

```
研究论文/报告
    ↓
① 先写入原文库（完整原文）
    ↓
② 再写入素材库（研究概述+索引指向原文库）
    ↓
陛下素材/笔记/洞察
    ↓
直接写入素材库（【我的笔记/思考】字段，原文保留）
    ↓
两库通过【原文索引】字段双向关联
    ↓
MASTER_INDEX.md（总索引表，统一入口）
```

---

## 七、执行要求（铁律）

1. **先写原文库，再写素材库**，顺序不能反
2. **每篇素材必须填原文索引**，索引错误或缺失视为不合格，退回重写
3. **陛下内容优先写入素材库**，不得以任何理由拒绝或延迟
4. **触发淘汰前必须先通知陛下**，得到确认后再执行
5. **所有研究必须先验证DOI真实性**再写入，不合格退回报酬
