# 日志生命周期管理

## 三级归档机制

### 热区（Hot Zone）
- **时间范围**：近7天
- **存储方式**：原始memory/YYYY-MM-DD.md文件
- **压缩**：不压缩
- **加载**：默认加载

### 温区（Warm Zone）
- **时间范围**：7天至30天
- **存储方式**：周摘要文件
- **文件位置**：memory/archive/weekly/YYYY-Wxx-summary.md
- **Token限制**：单文件≤2000 token
- **触发**：每周执行一次周总结归档
- **原始日志**：压缩为.tar.gz存入memory/archive/weekly/raw/

### 冷区（Cold Zone）
- **时间范围**：30天以上
- **存储方式**：月度/季度/年度摘要
- **Token限制**：
  - 月度摘要≤1000 token
  - 季度摘要≤500 token
  - 年度摘要≤500 token
- **触发**：每月执行一次月总结归档

---

## 双触发机制

### 时间触发
- 每周：周归档
- 每月：月归档

### 体积触发（先到先执行）
1. **热区日志文件数 > 7个** → 触发周归档
2. **热区单文件 > 50KB** → 触发即时摘要压缩
3. **温区周摘要文件数 > 4个** → 触发月归档
4. **memory目录总大小 > 10MB** → 触发全面归档清理
5. **MEMORY.md > 8000 token** → 触发MEMORY.md瘦身

---

## 记忆管家职责
- 每6小时执行一次巡检
- 统一管理所有agent的日志归档
- 执行压缩和摘要生成
