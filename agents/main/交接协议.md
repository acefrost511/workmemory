# 小艾 → IP主编 标准委托协议

**建立时间**：2026-03-27
**适用范围**：陛下向小艾下达内容创作任务时执行
**核心原则**：小艾只做调度，不做执行；执行交给IP主编

---

## 委托触发条件

以下任务，小艾必须委托给IP主编，不得自行执行：

1. 陛下要求生成洞察/创意洞察
2. 陛下要求撰写深度文章/公众号文章
3. 陛下要求做论文研究综述
4. 陛下转发素材/笔记，要求归档或整理
5. 陛下要求对已有内容做数据分析/复盘

以下任务，小艾可以自行处理：

1. 陛下日常聊天/问答
2. 陛下询问任务进度/状态
3. 陛下询问某个配置/规范内容
4. 陛下要求调整团队架构/规则
5.陛下询问今日工作进度

---

## 委托执行流程

### Step 1：小艾接收任务

- 理解陛下的需求
- 拆解为可执行任务
- 判断：自己执行还是委托IP主编

### Step 2：委托IP主编（sessions_spawn）

使用 sessions_spawn 工具，运行时为 agent_ip，传入以下信息：

```
任务类型：洞察生产/深度文章/素材归档/数据分析
规范文件：NORMS.md路径 + INSIGHT_LIBRARY.md路径
任务描述：[陛下原话或小艾整理后的任务描述]
质量要求：必须读取NORMS.md，执行全部质量检查
截止时间：陛下指定或有默认
```

### Step 3：小艾监控进度

- 每5分钟检查一次IP主编的执行状态
- 如有异常，立即上报陛下
- 如正常，等待IP主编完成

### Step 4：IP主编完成，汇报小艾

IP主编完成内容后：
- 存入指定文件（INSIGHT_LIBRARY.md / articles/ 等）
- 向小艾汇报完成状态
- 小艾向陛下推送最终产出

## 关键规范（委托时必须告知IP主编）

### 洞察生产规范（NORMS.md）

委托IP主编生成洞察时，必须强调以下铁律：

1. **批次查重**：读取INSIGHT_LIBRARY.md批次历史，确保本批次6篇文献与上一批次不重复
2. **批次内不重叠**：不同洞察之间最多重叠1篇，不允许两篇完全相同
3. **零虚构红线**：所有场景/人物/案例必须有真实来源，禁止虚构
4. **存储位置**：洞察存入INSIGHT_LIBRARY.md，格式见NORMS.md

### 素材归档规范

委托IP主编归档素材时：
- 笔记/案例 → 判断归属哪个信念抽屉 → 归入
- 无法归入 → 写入 staging_area.md → 等待陛下决策
- 归不进去也丢弃 → 直接丢弃，不保留

小艾委托IP主编时，统一使用以下话术框架：

```
【任务委托】

  - NORMS.md：/workspace/knowledge/insights/NORMS.md（必须先读）
  - 洞察库：/workspace/knowledge/insights/INSIGHT_LIBRARY.md（查批次历史）

质量红线（必检）：
  ① 批次内6篇文献无重复
  ② 零虚构：所有场景有真实来源
  ③ 每洞察至少2篇，不同洞察间最多重叠1篇
  ④ 达到100条时@小艾暂停
完成回报：向小艾汇报完成状态和产出路径
```
