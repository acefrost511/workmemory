# Token消耗优化策略

## 1. 懒加载策略（分层按需加载）

### 第一层：必加载（每次唤醒）
- **Token预算**：2000-3000 token
- **内容**：
  - SOUL.md
  - CONTEXT.md
  - 当前任务输入指令
- **加载时机**：每次唤醒必读

### 第二层：条件加载
- **Token预算**：0-5000 token
- **内容**：
  - SKILL.md（只加载相关技能段落）
  - HANDOFF.md（只在交接时加载）
  - 共享记忆（优先加载摘要索引）
- **加载时机**：根据任务类型决定

### 第三层：按需深挖
- **Token预算**：0-10000 token
- **内容**：
  - MEMORY.md
  - 近期日志/热区
  - 历史归档摘要
- **加载时机**：agent主动请求时

---

## 2. 两阶段提示（Two-Stage Prompting）

### 阶段一：轻量评估
- **Token预算**：约3000 token
- **操作**：加载第一层内容
- **输出**：执行计划（明确需要哪些额外资源）
- **预估完成率**：60%-70%日常任务

### 阶段二：完整执行
- **Token预算**：8000-15000 token
- **操作**：按需加载第二层+第三层
- **执行**：完成任务

---

## 3. 上下文硬限制

| 限制项 | 上限 |
|--------|------|
| 单次agent调用总上下文 | 40000 token |
| 单文件加载量 | 8000 token |
| 交接上下文 | 5000 token |
| MEMORY.md自身 | 10000 token |

### 触发压缩条件
- 超限时触发上下文压缩流程
- 超限时只取最新部分或加载摘要版本
- 超限时源agent必须自行压缩
- 超限时触发MEMORY.md瘦身
