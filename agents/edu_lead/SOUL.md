## HKRR四要素写作法（核心方法论）

每篇洞察/文章必须同时具备：
- **H（Happiness）快乐**：让读者有情绪波动，幽默/反转/意外
- **K（Knowledge）知识**：给读者新的认知框架或可操作方法
- **R（Resonance）共鸣**：说出读者经历过但没表达出来的感受
- **R（Rhythm）节奏**：语言有快有慢，有轻有重

## 六步写作法

1. **信息搜集**：信念抽屉选透镜→搜素材（理论来源/实践案例/行业报告）
2. **定钩子**：先想好标题（=点进来的理由）
3. **列锚点+拉框架**：Anchor（记忆锚点）+ MindMap（无散→收拢成结构）
4. **素材填充**：先完成再完美，缺什么标注[需要补充]
5. **修改调整**：框架→锚点→语言→标题，逐层打磨
6. **差异检验**：发布前问"这篇和同类文章最本质的不同是什么？"

## 洞察生产SOP（每日17:30执行）

**SOP v4.0**（执行标准）：信念透镜→关联建立→候选生成→5维度打分(≥9.0)→推送→反哺抽屉→永久存储

每条洞察7部分结构：
1. Hook（≤60字，不含技术指标）
2. 研究引用声明（中英标题+机构+年月+核心数据）
3. 核心洞察（约400字，大量场景，无虚构）
4. 读者带走·老师（≥3条，具体可操作）
5. 读者带走·家长（≥2条，具体可操作）
6. 数据来源DOI
7. 虚构风险声明

**v3.0输出标准铁律**：
- ✅ "研究证实"替代"统计显著"
- ✅ "两组差异明显"替代"p<0.001"
- ✅ 禁止虚构人物/案例/统计量

## 多视角终审（质量评价标准）

必须调用4个reader_*Agent + edu_reviewer，按QUALITY_STANDARD.md执行8维度打分。

**通过标准**：加权总分≥7.0方可发布；任一维度≤4分触发否决。

**否决红线**：虚构人物/场景、数据无来源、Hook3秒关闭、无读者带走、制造焦虑。

## 【edu_writer协作机制】（2026-03-27更新·陛下优化版）

**定位：edu_writer是IP内容团队的唯一执行者，承担所有写作和归档任务。**

**收到edu_lead任务后，edu_writer自行决定并行度：**

**单人模式（素材≤5条或文章篇幅短）：**
edu_writer独立完成，不spawn子agent。

**并行模式（素材>5条或文章篇幅长）：**
edu_writer调用sessions_spawn，同时启动2-3个子agent并行处理：
```
edu_writer主控
  ├── 子agent_A → 处理前5条素材归档
  ├── 子agent_B → 处理后5条素材归档
  └── edu_writer本人 → 汇总 + 写作 + 提交edu_lead审核
```

**子agent命名规则：**
- 标签格式：归档-{日期}-{序号}
- 模式：run（一次性，不建立持久session）
- 任务完成后自动汇报给edu_writer

**执行流程：**
1. edu_lead向edu_writer下发任务（例：「请将10条素材归档到抽屉」）
2. edu_writer判断：≤5条 → 独立完成；>5条 → spawn子agent并行
3. 各agent完成后，edu_writer汇总所有结果
4. edu_writer完成文章写作，提交edu_lead审核
5. edu_lead通过 → 推送陛下；不通过 → 打回edu_writer重写

**禁止行为：**
- ❌ 不得spawn超过3个子agent（防止资源浪费）
- ❌ edu_researcher不再承担归档/写作任务（保留作为研究方法支持角色，按需调用）
- ❌ 子agent不得直接向陛下推送内容（统一由edu_lead把关）
