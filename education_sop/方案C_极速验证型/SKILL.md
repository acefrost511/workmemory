# 方案C：极速验证型

## 核心流程
```
素材 → 轻量提示词写作(v1) → 资深教师评审 → 快速修改 → 呈报
```

## 方案特征
- **风格**：快速、简洁、直接
- **优势**：流程短（60-80分钟/篇），陛下介入少
- **劣势**：少了新手视角和主编把关，质量略低但速度优先

## 核心原则

### 轻量提示词（单次写作）
所有信息一次给到位，不需要多轮迭代：
1. 主题
2. 读者定位（新手教师/资深教师）
3. 核心观点（1句话）
4. 核心数据（2-3个关键数字）
5. 文章类型（叙事/分析/观点）
6. 字数要求（1500字）

### 单刀直入写作法
- **开头3句**：必须进场景，不能寒暄
- **每段结构**：观点句+例子，不绕弯子
- **数据引用**：必须给出参照系（大小/高低/快慢）
- **结尾**：金句+最小行动（读者现在能做什么）

## 快速评审清单

资深教师用以下清单快速评审（10分钟内完成）：
1. [ ] 有没有具体学校/学科/年级场景？
2. [ ] 核心数据有没有参照系？
3. [ ] 有没有AI失误的诚实说明？
4. [ ] 有没有一个金句能独立传播？
5. [ ] 读者现在能做什么，说清楚了吗？

## 评分体系（单维度）
- 场景真实感(1-10)
- 数据说服力(1-10)
- 实操指引清晰度(1-10)
- 诚实度（有没有说AI的问题）(1-10)
- 传播潜力(1-10)
满分50分，折算百分制

## 输出要求
- 字数：1200-1500字
- 格式：Markdown
- 特点：简洁直接，适合陛下快速浏览后微调发布
