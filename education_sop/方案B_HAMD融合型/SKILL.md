# 方案B：HAMD融合型

## 核心流程
```
触发判断 → Hook定题 → Anchor定锚 → MindMap发散 → Different定位 → edu-editor写作 → 资深教师评审
```

## HAMD四步法（写前必做）

### Step 1: Hook - 这篇文章要回答什么问题
- 不是"写什么主题"，是"读者读完后能带走什么"
- 写出Hook问题（一个具体的、读者真正想知道答案的问题）
- 示例："AI批改作业真的靠谱吗？一位老师的真实体验告诉你"

### Step 2: Anchor - 读者读完能记住哪句话
- 找出文章的核心判断（一个简单、清晰、有力的句子）
- 这句话要经得起推敲，能独立传播
- 示例："AI是放大镜，教师是方向盘"

### Step 3: MindMap - 八维发散（写前不筛选）
八个维度：
1. 数据/研究：相关研究数据、数字
2. 案例/故事：具体教学场景、学生反应
3. 痛点/问题：读者的真实困惑
4. 方法/技巧：具体可操作的做法
5. 边界/局限：什么时候不适用
6. 对比/反例：与同类做法相比的优势/劣势
7. 情绪/共鸣：让读者产生情感连接的元素
8. 行动/建议：读者现在可以做什么

**规则**：先无序列出所有想到的，不筛选，不评判

### Step 4: Different - 和同类文章比最本质的不同
- 列出3-5篇同类爆款文章
- 找出自己的文章最独特的一个角度
- 这个角度要足够小、足够深，不能泛泛

## 写作执行

### 标题创作（Hook的具体化）
- 标题公式：[具体场景/数字] + [核心洞察] + [悬念/情绪]
- 避免：泛泛的"AI+教育"类标题
- 目标：读者一看标题就想点进来

### 结构选择（根据文章类型）
- 场景切入式：开头用真实场景，抓住注意力
- 认知翻转式：先说流行观点，再说真实情况
- 问题答疑式：用读者问题串起全文
- 故事线式：用叙事线串联全文

### 填充写作（先填进去，不管对错）
- 按框架顺序写，不要跳段
- 写得不好也要写完，不要卡住
- 每一段：观点 + 例子的顺序

### 修改四问（写完后自检）
1. 框架：每个模块是否服务Anchor？
2. 锚点：核心金句是否有力、能否独立传播？
3. 语言：术语是否都着陆了？节奏是否流畅？
4. 标题：是否比原来更好了？

## 评分体系（简化版）
- Hook吸引力(1-10)
- Anchor传播力(1-10)
- 内容深度(1-10)
- 实操价值(1-10)
- 场景适配(1-10)
- 标题质量(1-10)
满分60分，折算百分制

## 输出要求
- 字数：1500-2000字
- 格式：Markdown
- 必须包含：Hook问题、Anchor金句、Different定位说明
