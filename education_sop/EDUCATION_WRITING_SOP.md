# 教育内容创作SOP框架 v2.0

_整合两轮测试优化，2026-03-24_

---

## 一、整体架构

```
素材库 ──→ 主题库 ──→ 写作 ──→ 多agent评审 ──→陛下审稿 ──→ 排版 ──→ 发布
              ↑                              ↓
          陛下审批                   修改后再进入排版（重要！）
```

---

## 二、素材库

### 来源A：陛下主动添加
- 手动存入素材库，直接进入「待写区」，无需审批

### 来源B：情报官检索
- 情报官(info_officer)每天检索10路K12教育资讯
- 提交审核列表 → 陛下审批 → 入库
- 分级规则：
  - 高可信来源（学术期刊/Sci-Hub/DOAJ等）→ 可设置「免审直入」
  - 一般来源（新闻/公众号/报告）→ 必须审批

## 三、主题精选库

陛下持续完善，分六大类：
- 论文解读类（基于学术研究的解读）
- 实践分享类（真实教学经验，干货实用）
- 政策分析类（教育政策解读）
- 产品评测类（教育产品对比）
- 资源推荐类（工具、课程推荐）
- 综合评论类（观点输出）

## 五、工作流程（详细6步）

### Step 1：素材入库
- 情报官提交今日审核列表
- 陛下审批哪些进入素材库
- 或陛下直接添加素材

### Step 2：选题
- 主编从主题库挑选写作方向
- 结合素材库中的热点素材
- 陛下确认选题

### Step 3：写作调度
- 主编根据主题类型，分发到对应写作技能
- 生成初稿（v1）
- 标注字数、类型、核心观点

### Step 4：多agent评审（重要！v1必须先过评审再给陛下）
- 主编自评（事实准确性、逻辑连贯性、主题契合度）
- 资深教师agent评审（专业深度、实操价值、场景适配、专业规范）
- 如v1评分<8分 → 主编根据评审意见优化 → 生成v2 → 再次评审
- 评分标准：综合得分≥8分才能进入陛下审稿环节

### Step 5：陛下审稿
- 主编推送评审后的定稿给陛下
- 陛下提出修改意见或通过
- **必须通过评审再进排版，避免白做！**

### Step 6：排版+发布
- 调用md2wechat技能
- 将markdown转成微信公众号HTML
- 直接粘贴到公众号编辑器

## 六、Agent角色定义

| Agent | 角色 | 职责 |
|-------|------|------|
| 情报官 | 素材收集 | 检索、筛选、提交审核 |
| K12教育专家 | 专业评审 | 评估学术价值、事实准确性 |
| 主编 | 统筹调度 | 选题、分发任务、审初稿、做主编自评 |
| 资深教师agent | 第一读者 | 评分专业深度、实操价值、场景适配 |
| 新手教师agent | 第二读者 | 评分可读性、吸引力（对比视角） |
| 写作agent | 内容生成 | 执行各类型文章写作 |

## 七、评分体系 v2.0（已优化）

### 资深教师agent评分维度（5个）
1. 专业深度（1-10）：理论支撑、概念准确性
2. 实操价值（1-10）：方法是否可落地
3. 场景适配（1-10）：是否考虑农村/薄弱校等不同场景
4. 专业规范（1-10）：术语准确、逻辑严密
5. **可操作性（1-10）**：读者能否直接复制方法，有无配套清单

### 新手教师agent评分维度（3个）
1. 可读性（1-10）：语言流畅、结构清晰
2. 吸引力/标题（1-10）：标题是否有网感
3. 阅读体验（1-10）：排版、段落长度

### 综合得分计算
```
最终分 = 资深教师综合×50% + 新手教师综合×30% + 主编自评×20%
```

### 通过门槛
- ≥8分 → 进入陛下审稿
- 7-8分 → 主编优化后复评
- <7分 → 重新分析素材、调整主题、重写

## 八、写作铁律（edu-editor + 实践类专项）

### 通用铁律（所有类型）
1. 概念不删，但必须配"翻译"（首现术语要解释）
2. 每个抽象论点紧跟可落地的教学例子
3. 数据和来源是硬通货（注明来源、保留原始数字）
4. 开头"破题"而非"寒暄"（3句内进场景）
5. 结尾必须有独立传播力（金句+可执行行动）
6. 语气"尊重型对话"，不居高临下
7. 标题和小标题传递实质信息

### 实践叙事类专项铁律
1. 开头要"痛"，不要寒暄（3句内进场景）
2. 数据要"真"，不要堆砌（精确分钟数+参照系）
3. 惊喜有画面感，烦恼有清单感（最强差异化）
4. 结尾有金句，更要有行动（附可直接复制的工作流程）
5. 术语要"着陆"，每千字不超过5个，首现配解释

### AI/技术工具类文章专项铁律
1. 必须包含AI典型失误类型说明
2. 必须有教师快速核查方法
3. 必须标注不可省略的人工环节（"复核清单"）
4. 涉及不同学校条件时，明确说明文章适用场景

## 九、迭代优化规则

1. v1生成后，**必须先经多agent评审**，不达标不能给陛下看
2. 评审→优化→再评审→通过后才能进入陛下审稿
3. 每轮测试记录评分和主要问题，沉淀到本文档
4. SOP版本号按测试轮次更新

## 十、测试记录

### 第一轮测试（论文解读类）
- 测试素材：AI辅助教学元分析（58项研究）
- v1评分：资深教师4.0/5.0（折算8.0/10）
- 主要问题：缺农村/薄弱校视角；行动建议颗粒度不够
- 改进：已补充场景适配维度

### 第二轮测试（实践分享类）
- 测试素材：AI批改作业真实体验
- v1评分：88分 → v2评分：92分
- 主要发现：叙事感染力强；但需专项模板
- 改进：已新增实践叙事类写作铁律和AI工具专项规则
