# 教育内容创作SOP框架 v3.0（最终版）

_整合三轮测试全部发现，2026-03-24 通宵测试完成_

---

## 一、整体架构（最终版）

```
素材库 ──→ 主题库 ──→ 写作 ──→ 多agent评审（必须！） ──→陛下审稿 ──→ 排版 ──→ 发布
              ↑                              ↓
          陛下审批                 修改后再进排版，避免白做
```

**核心原则：v1必须先过多agent评审，不达标绝不直接给陛下。**

---

## 二、素材库

### 来源A：陛下主动添加
- 手动存入素材库，直接进入「待写区」，无需审批

### 来源B：情报官检索
- 情报官每天检索10路K12教育资讯
- 提交审核列表 → 陛下审批 → 入库
- 分级规则：
  - 高可信来源（学术期刊/Sci-Hub/DOAJ等）→ 可设置「免审直入」
  - 一般来源（新闻/公众号/报告）→ 必须审批

## 五、工作流程（最终版6步）

### Step 1：素材入库
情报官提交审核列表 → 陛下审批入库，或陛下直接添加

### Step 2：选题
主编从主题库挑选方向 → 陛下确认选题

### Step 3：写作调度
主编按类型分发 → 写作agent生成v1 → 标注字数/类型/核心观点

### Step 4：多agent评审（关键步骤！）
```
v1生成
  ↓
资深教师agent评审 → 输出评分+修改意见
  ↓
新手教师agent评审 → 输出评分+修改意见（术语友好度重点检查）
  ↓
主编汇总 → 计算加权综合分
  ↓
≥8分 → 进入Step5
  7-8分 → 主编按意见优化 → v2 → 再次评审 → 通过后进Step5
  <7分 → 重新分析素材 → 调整主题 → 重写
```

### Step 5：陛下审稿
主编推送评审通过稿 → 陛下终审 → 通过后进排版

### Step 6：排版+发布
md2wechat转换 → 粘贴到公众号编辑器 → 发布

## 六、Agent角色定义

| Agent | 角色 | 核心职责 |
|-------|------|---------|
| 情报官 | 素材收集 | 检索、筛选、提交审核 |
| K12教育专家 | 专业评审 | 评估学术价值、来源可靠性 |
| 主编 | 统筹调度 | 选题、分发任务、汇总评审、主编自评 |
| 资深教师agent | 第一读者 | 评分专业深度/实操价值/场景适配/专业规范/可操作性（5维度） |
| 新手教师agent | 第二读者 | 评分可读性/吸引力/阅读体验（3维度）；**术语障碍一票否决** |
| 写作agent | 内容生成 | 执行各类型写作 |

## 七、评分体系 v3.0（最终版）

### 资深教师agent评分（5维度，满分50分）
1. 专业深度（1-10）：理论支撑、概念准确、批判性分析
2. 实操价值（1-10）：方法是否可落地、步骤是否清晰
3. 场景适配（1-10）：是否考虑农村/薄弱校/不同学段/学情差异
4. 专业规范（1-10）：术语准确、逻辑严密、无知识性错误
5. **可操作性（1-10）**：读者能否直接复制、有无配套清单

### 新手教师agent评分（3维度，满分30分）
1. 可读性（1-10）：语言流畅、结构清晰、无术语断层
2. 吸引力/标题（1-10）：标题是否有网感、开头是否能抓住注意力
3. 阅读体验（1-10）：段落长度、节奏感、排版友好度

### 主编自评（2维度，满分20分）
1. 主题契合度（1-10）：是否紧扣选题方向
2. 传播潜力（1-10）：标题/结尾/金句是否有传播力

### 综合得分计算
```
论文解读类：资深×35% + 新手×25% + 主编×20% = 满分100
实践分享类：资深×25% + 新手×45% + 主编×20% = 满分100
政策分析类：资深×35% + 新手×35% + 主编×20% = 满分100
产品评测类：资深×30% + 新手×35% + 主编×20% = 满分100
资源推荐类：资深×25% + 新手×40% + 主编×20% = 满分100
```

### 通过门槛（双重底线）
1. **综合底线**：加权总分 ≥ 80分
2. **新手教师底线**（安全阀）：新手教师得分 < 总分33%（即<10/30分）→ 强制返工

### 评级
- 90-100：优秀，可直接发布
- 80-89：良好，陛下微调后可发布
- 70-79：一般，主编优化后复评
- <70：较差，重新写作

## 八、写作铁律（完整版）

### 通用铁律（所有类型）
1. 概念不删，但必须配"翻译"（首现术语→口语化解释+具体场景）
2. 每个抽象论点紧跟可落地的教学例子（学科/年级/课堂动作）
3. 数据和来源是硬通货（保留原始数字，注明来源机构）
4. 开头"破题"而非"寒暄"（3句内必须进场景）
5. 结尾必须有独立传播力（金句+可执行最小行动）
6. 语气"尊重型对话"，不居高临下
7. 标题和小标题传递实质信息（好小标题=微型论点）
8. **术语首次出现必须括号注明含义**，尤其是缩写（如ZPD即最近发展区）

### 实践叙事类专项铁律
1. 开头要"痛"，不要寒暄（3句内进场景，用具体数字制造共鸣）
2. 数据要"真"，不要堆砌（精确分钟数+参照系）
3. 惊喜有画面感（具体学生行为），烦恼有清单感（检查清单）
4. 结尾有金句，更要有行动（附可直接复制的工作流程）
5. 术语要"着陆"，每千字不超过5个，首现必配解释

### AI/技术工具类文章专项铁律
1. 必须包含AI典型失误类型（举2-3个具体例子）
2. 必须有教师快速核查方法（5分钟内可完成的检查步骤）
3. 必须标注不可省略的人工环节（"复核清单"）
4. 涉及不同学校条件时，明确说明文章适用场景（农村/薄弱校需单独说明）

## 九、三轮测试完整记录

### 第一轮：论文解读类
- 素材：AI辅助教学元分析（58项研究，12000+学生）
- v1评分：7.5-8.0分 → v2评分：**9.0/10分**（一轮优化即通过）
- v1→v2主要改进：补充效应量参照系、行动建议量化、场景边界说明、学科内部分类、新增常见误区板块
- **额外发现（已沉淀为SOP规则）**：
  1. 数据参照系规则：引用效应量必须提供惯常分类（d=0.2小/0.5中/0.8大）
  2. 行动建议量化：每条建议必须含频率+评估指标+试点规模
  3. 学科内部分类：学科差异分析必须做内部分类，禁止一刀切
  4. 场景边界说明：文章必须诚实说明适用范围（城镇/农村等）
  5. 常见误区知识库：主编审核时应检查是否有失败模式警示

### 第二轮：实践分享类
- 素材：AI批改作业真实体验（深圳某小学）
- v1评分：88分 → v2评分：92分
- 发现：叙事感染力强；需专项叙事模板
- 改进：新增实践叙事类5条铁律；新增"可操作性"评分维度

### 第三轮：评分体系对比
- 素材：课堂提问技巧（约1200字）
- 发现：资深教师8.0/10 vs 新手教师6.0/10（系统性差异，非偶然）
- 三个核心发现：
  1. 术语障碍是最大分歧点（新手遇到ZPD等缩写产生认知断层）
  2. 不同类型文章需要不同读者权重
  3. 新手教师得分<33%具有"一票否决"价值
- 改进：建立6类文章差异化权重矩阵；新增双重底线规则
