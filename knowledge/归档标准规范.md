# 归档标准规范
> 版本：v1.0 | 制定：2026-03-27 | 维护者：小艾
> **所有参与归档的Agent在执行归档前必须读取本文件**

---

## 一、归档流程总览

```
情报官扫描完成 → 生成当日活跃素材库 → 通知IP主编
                                           ↓
IP主编分发任务 → 多Agent并行归档 → 各Agent追加到对应信念抽屉
                                           ↓
                                   完成后汇报给IP主编
                                           ↓
                                   IP主编汇报给小艾 → 小艾通知陛下
```

---

## 二、活跃素材库文档格式

**命名**：`素材库_YYYY-MM-DD.md`
**位置**：`/workspace/knowledge/素材库/`
**每日生成一份，当日结束时不再修改**

**单条素材格式**：
```
【素材001】
收录时间：YYYY-MM-DD HH:mm
类型：研究论文 / 国际报告 / 政策文件 / 陛下素材
状态：待归档 / 已归档

【标题】
英文：[原文]
中文：[翻译]

【基本信息】
作者：XXX
机构：XXX
期刊/来源：XXX
发表时间：YYYY年MM月
DOI：10.XXXX/XXXXX（或"无"）
原文索引：/workspace/knowledge/原文库/XXX.md

【研究背景】（≥400字）

【研究发现】（≥400字）

【对教育的价值/启示】（≥200字）

【我的笔记/思考】（陛下素材原文，不足1000字原文保留，超1000字压缩）
```

**归档判断优先级**：
1. 先看标题主题（排除明显不相关的）
2. 再看研究背景和研究发现（判断核心主题）
3. 一个素材只归属一个抽屉，选择最相关的一个
4. 无法判断 → 放入最相关的"候选抽屉"，标注[待确认]
5. 陛下素材 → 优先看【我的笔记/思考】的关键词判断抽屉

## 四、多Agent协作归档规则

**参与归档的Agent**：
- edu_lead（主编·总负责）
- edu_writer（编辑·协作归档）
- edu_researcher（研究员·协作归档）

**分发机制**：
- IP主编(edu_lead)收到通知后，计算当日素材数量
- 素材数量≤5条 → edu_lead单独完成
- 素材数量>5条 → edu_lead分配给edu_writer和edu_researcher协作
- 每人负责若干条，独立判断抽屉归属

**协作格式（在当日素材库中标注分工）**：
```
【素材001】→ edu_writer负责
【素材002】→ edu_researcher负责
【素材003】→ edu_lead负责
```

## 五、归档执行标准

**每条素材的归档操作**：
1. 在活跃素材库中将该条状态改为：`✅ 已归档`
2. 追加到对应信念抽屉文档末尾
3. 格式如下：

**追加到信念抽屉的格式**：
```
---
【归档素材】
来源：素材库_YYYY-MM-DD.md（素材001）
类型：研究论文 / 陛下素材
归档时间：YYYY-MM-DD HH:mm
归档人：XXX

【核心观点】
（一句话概括本素材的核心论点）

【详细内容】
（完整粘贴研究背景+研究发现+对教育的价值+陛下的笔记/思考）

【关联旧素材】
（如有相关旧素材在此抽屉，标注关联，如：关联信念1第3条、信念3第7条）
```

## 六、归档质量要求

**归档质量检查**：
- ✅ 每条素材是否正确归入最相关的抽屉
- ✅ 抽屉文档末尾是否追加了新内容
- ✅ 活跃素材库中状态是否更新为"已归档"
- ✅ 内容是否完整（不能截断，必须完整粘贴）
- ⚠️ 无法判断抽屉 → 标注[待确认] + 通知IP主编决定

**禁止行为**：
- ❌ 将素材同时放入多个抽屉（一物多放）
- ❌ 只粘贴标题，不粘贴正文内容
- ❌ 未读正文仅凭标题判断抽屉归属
- ❌ 删除或覆盖抽屉中已有的旧素材

## 八、特殊情况处理

**素材无法归入任何抽屉时**：
→ 暂存到「待归类素材」文件夹
→ 通知陛下判断归属哪个抽屉
→ 陛下决定后补充规则到本规范

**某抽屉内容过多时**：
→ 当前抽屉文档超过500KB时 → 拆分历史版本到历史库
→ 格式：`信念X_v历史日期.md`

**活跃素材库超30天未归档**：
→ 每月最后一天执行批量归档
→ 完成后将旧活跃库移入历史库
→ 格式：`历史库/素材库_YYYY-MM-DD_已归档.md`
