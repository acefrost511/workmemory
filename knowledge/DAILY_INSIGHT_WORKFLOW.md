## 一、这套系统做什么

每天17:30，从三个数据源（信念抽屉、笔记库、素材库）里提取内容，
找出2-3个信念之间的碰撞点，生成3条面向K12老师的洞察，推送给陛下。

## 四、完整执行流程（Step by Step）

### Step 1：主编读取文件（edu_lead）

主编依次读取以下文件，建立今日洞察方向：

1. **/workspace/knowledge/DAILY_INSIGHT_WORKFLOW.md**（本文档）
   → 确认今日工作流程
2. **/workspace/knowledge/IP_WORKFLOW_V1.md**
   → 确认权重机制、达标条件、输出格式
3. **信念抽屉（2-3个）**
   → 从信念1-13中选2-3个，读取每个信念的：
     - 核心表述+金句
     - 素材积累区的研究列表

主编如何选择信念：
- 看今日素材库有无新增研究，追溯到对应信念
- 看最近哪几个信念之间有内在联系（如信念2讲教师角色+信念6讲课堂教学）
- 看陛下最近有无新的笔记，涉及哪些主题

### Step 2：主编确定碰撞主题

碰撞的本质：找两个/三个信念的素材之间的逻辑关系

碰撞类型：
- **印证型**：A研究+B研究共同指向同一结论（可信度高）
- **矛盾型**：A研究和B研究结论不同（矛盾处=洞察入口）
- **互补型**：A研究补充了B研究忽略的视角

禁止行为：
- 不能随意选两个信念，必须有逻辑关联
- 不能只看标题选信念，必须读素材内容

### Step 3：主编spawn edu_writer

主编spawn edu_writer，告知：
- 选定哪2-3个信念（提供信念编号和名称）
- 碰撞主题是什么

**edu_writer 接到任务后，必须：**

1. 读取 /workspace/knowledge/INSIGHT_WRITING_RULES.md（写作规范·铁律）
2. 读取对应信念抽屉文件（核心表述+素材积累区）
3. 如需参考陛下笔记：读取 /workspace/knowledge/笔记/ 下对应文件

**edu_writer 产出3个候选洞察，每篇格式：**

```
【洞察编号】
Hook（≤60字，必须含数字或反常识）

引用证据A
英文标题（中文翻译）
作者+机构+期刊+年月+DOI（必须完整，缺一不可）
核心发现（原文数字，不改写）

引用证据B
英文标题（中文翻译）
作者+机构+期刊+年月+DOI
核心发现（原文数字，不改写）

碰撞逻辑（必须只引用A研究×B研究的逻辑关系，禁止混入任何虚构人物/场景）

读者带走·老师（≥3条，每条≥30字）
读者带走·家长（≥2条，每条≥30字）

虚构声明（必须写）：本洞察无虚构内容，所有数据来自原文研究。
```

**edu_writer 自检清单（写完必查）：**
- [ ] Hook≤60字？
- [ ] 每篇含完整DOI（英文标题+作者+机构+期刊+年月）？
- [ ] 碰撞逻辑只引用了研究，没有虚构人物/学校？
- [ ] 所有数字与原文一致？
- [ ] 读者带走老师≥3条，家长≥2条？

### Step 4：主编并发spawn 6个读者Agent

收到 edu_writer 的3个候选洞察后，主编同时spawn 6个读者Agent：
- reader_new_teacher
- reader_senior_teacher
- reader_principal
- reader_parent
- reader_expert

每个读者Agent：
1. 读取 edu_writer 的洞察草案
2. 根据自己的视角独立评分
3. 完成后 sessions_send 回报主编

**每个读者Agent的评分标准：**
- 虚假内容核查（最优先）：核心洞察是否含虚构人物/学校/场景？若有，直接扣分至≤4
- 学术严谨性：DOI是否完整？数字是否与原文一致？
- 读者带走实用性：读者看完知道该怎么做吗？
- 认知增量：读者有新收获吗？

### Step 5：主编汇总评分

主编收集6份评分报告，计算加权总分。

**达标条件（必须同时满足）：**
- ① 加权总分 ≥ 9.0
- ② 无单维度得分 ≤ 4
- ③ 社交货币指数 ≥ 6
- ④ 文章传播力 ≥ 6

**判断结果：**
- 全部达标 → 进入 Step 6（终审）
- 未达标 → 打回 edu_writer，说明具体修改意见，最多3次迭代
- 3次迭代仍<9.0 → 放弃该洞察，换候选洞察

### Step 6：主编终审（最重要的防线）

主编必须逐项核查：

**虚构内容检查（最优先）：**
- [ ] 核心洞察段落含虚构人物（"某老师""某学生""某学校"）？
- [ ] 碰撞逻辑段落含虚构场景？
- [ ] 数据是否与原文一致，有没有为了"听起来合理"而添加的数字？
- [ ] 引用证据的DOI是否可验证？

**格式检查：**
- [ ] Hook≤60字？
- [ ] 读者带走老师≥3条、家长≥2条？
- [ ] 每篇洞察含完整DOI？

**发现虚构内容** → 立即打回 edu_writer，不推送
**发现DOI缺失** → 打回，不推送
**全部核实无误** → 进入 Step 7

### Step 7：反哺抽屉（exec写入，不用edit）

洞察通过后，主编执行以下命令：

```
# 写入洞察记录
洞察编号/日期/总分/社交货币/传播力 >> /workspace/knowledge/beliefs/信念{N}-{名}.md
```


## 五、绝对禁止事项（违者打回/终止）

- ❌ 编造任何具体人物/学校/场景/情节
- ❌ 在"碰撞逻辑"段落混入虚构内容
- ❌ 修改研究原文的数字
- ❌ 以"某学生""某老师""某学校"作为论证依据
- ❌ 虚构案例（未注明来源的"真实场景"）
- ❌ 引用无DOI的论文（国际期刊必须DOI，CNKI论文除外）

**唯一合法场景来源**：陛下笔记（/workspace/knowledge/笔记/）

**如果素材不足——只写研究数据，不写场景。宁可单薄，不可虚构。**

## 六、输出格式模板

每篇洞察的完整格式如下（edu_writer按此模板写作）：

```
【洞察编号】
标题：（≤20字，概括核心观点）

Hook（≤60字，必须含数字或反常识）

━━━━━━━━━━━━━━━━━━

引用证据A
英文标题：（原文英文标题，中文翻译在括号内）
中文译名：（中文翻译全称）
作者：（作者全名）
机构：（研究机构）
期刊：（期刊名称）
发表年月：（YYYY年MM月）
DOI：（完整DOI，例：10.1007/s10639-025-13393-z）

核心发现：（原文主要结论，不改写数字）

━━━━━━━━━━━━━━━━━━

引用证据B
[同上格式]

━━━━━━━━━━━━━━━━━━

碰撞逻辑：
（A研究的核心发现 + B研究的核心发现 → 两者共同指向什么结论？交汇点在哪里？为什么这个交汇点重要？）

━━━━━━━━━━━━━━━━━━

核心洞察：
（基于上述两研究的逻辑推导，得出核心洞察，350-450字）

━━━━━━━━━━━━━━━━━━

读者带走·老师（≥3条，每条≥30字）：
① （具体可执行的操作建议）
② （具体可执行的操作建议）
③ （具体可执行的操作建议）

读者带走·家长（≥2条，每条≥30字）：
① （具体可操作的家庭建议）
② （具体可操作的家庭建议）

━━━━━━━━━━━━━━━━━━
虚构声明：（必须写）本洞察无虚构内容，所有数据和发现均来自原文研究，可查DOI验证。
洞察信念溯源：信念X（信念名称）× 信念Y（信念名称）
综合评分：X.XX/10
```
