# 笔记库 · 处理流水线

## 一、入口优化

### 录音入口
陛下直接发语音/录音 → 臣自动转文字 → 存入raw_content → 清洗存入processed_content

### 文本入口
陛下粘贴文字片段 → 臣判断是否为笔记素材 → 存入raw_content → 清洗存入processed_content

### 文章/对话入口
陛下发链接或内容 → 臣提炼核心观点 → 存为笔记 → 标注来源，保留原文引用

---

## 二、处理流水线（臣自动执行，陛下触发后5分钟内完成）

```
陛下输入（语音/文本）
    ↓
自动转写（若为语音）
    ↓
第一步：保存原文
    raw_content字段：一字不改保存原始输入
    仅去除明显口语填充词（语气词、重复开头），原意不变
    ↓
第二步：清洗正文
    processed_content字段：
    - 删除重复语句、填充词（"那个"、"然后"、"其实"等）
    - 修正AI教育行业口误
    - 合并断句，还原完整语义
    ↓
第三步：结构化提炼
    - 提炼标题（≤50字）
    - 提炼一句话摘要（≤100字）
    - 抽取关键词（3-5个）
    - 识别关联主题标签
    ↓
第四步：与精选主题库匹配（核心新增）
    读取 /workspace/knowledge/topic_library.md
    计算每条笔记与每个精选主题的匹配得分
    匹配维度：核心关键词重叠度、观点契合度、应用场景一致性
    存入 matched_topics 字段（matching_score 0-1）
    ↓
第五步：笔记间关联
    与已有笔记关联（主题/关键词/观点相似度）
    更新 related_notes 字段
    ↓
第六步：存入笔记库 + 更新索引
    notes_index.json 同步更新 matched_topics 和 related_notes
    ↓
通知陛下（可选，简短提示）
```

## 三、与主题精选库的匹配机制

### 匹配触发条件
每条笔记处理时自动触发，无需陛下指令

### 匹配流程
1. 读取当前精选库所有主题（topic_library.md）
2. 对每条笔记，逐个计算与每个主题的匹配得分
3. 得分≥0.3的匹配项全部记录到matched_topics
4. 得分最高的匹配项作为主要关联

### 匹配得分含义

| 得分 | 含义 | 处理方式 |
|------|------|----------|
| 0.9-1.0 | 直接命中已有主题 | 标记为该主题的核心素材 |
| 0.6-0.89 | 部分匹配 | 作为已有主题的补充素材 |
| 0.3-0.59 | 相关领域 | 纳入通用知识库，可作为新主题候选 |
| < 0.3 | 无关联 | 单独保存，等待未来匹配 |

### 匹配示例
```
matched_topics:
  - topic_title: "AI在K12课堂的实际效果"
    matching_score: 0.87
    match_reason: "笔记中提到的AI批改数据与该主题直接相关"
  - topic_title: "教师AI素养培养"
    matching_score: 0.42
    match_reason: "涉及教师使用AI工具的态度，但非核心论点"
```

## 四、笔记→文章判断标准（更新）

- matched_topics中最高matching_score ≥ 0.6
- matched_topics中存在matching_score ≥ 0.9的笔记≥2条
- 有至少1个可量化数据/案例支撑
- 臣主动向陛下建议："X个笔记已匹配到'XXX'主题（均分0.85），建议优先写作"

---

## 五、效率承诺

- 陛下发来内容 → 5分钟内完成全流程
- 笔记达到3条同主题且matching_score均值≥0.6 → 自动建立"主题束"
- 每晚20:00洞察推送时，读取笔记的matched_topics字段，无需重新计算
