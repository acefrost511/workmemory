# Harness Engineering：给AI这匹野马套上缰绳

> 来源：book2skills.com 创始人实践分享

## 一、核心概念解析

**Harness Engineering 寓意**
- Harness 原意：缰绳和马鞍
- 寓意：AI这匹马很强大，但没有缰绳和马鞍，它就只会乱跑
- 构建者 = 骑手 = 写"秘籍"的人

**为什么这个词在这个时代出现**
- AI能力已经足够强，但可靠的应用系统没有跟上
- 本质问题：从"让AI做一件事"到"让AI可靠地持续做一系列事"
- 解决方案：提前写好标准、原则、检验标准、Skills——这就是"秘籍"

---

## 二、AI Engineering三阶段（纵向对比）

**阶段一：Prompt + Workflow**
- 单次调用 + 硬性编排
- 流程固定，灵活性差
- 典型：简单问答、固定模板

**阶段二：上下文工程**
- 支持长步骤推理和灵活编排
- 但过程中容易跑偏，需要人在旁边盯着
- 典型：Coze工作流、复杂Agent编排

**阶段三：Harness Engineering**
- 无人值守运行
- 核心：提前写好"秘籍"（文档+标准+Skills+反馈循环）
- AI遇到问题自己去查这些文档，不是等人类在旁边纠正
- **这是进化的终点**：人从"在旁边盯着"变成"设计完就去喝茶"**

## 三、Book2Skills案例：如何起步Harness Engineering

**花了多少时间**：2天，4个步骤

**步骤一：充分的Requirement编写（2小时）**
- 先花时间把需求想清楚
- 逼成明确的文字标准
- 产出：requirements.md

**步骤二：构建合理大小的Skill（5小时，最核心）**
三个原则：
1. 先粗糙跑通（最简单输入，输出最基础版本）
2. 逐个精细化（一点点提升质量）
3. 流程拆解（超过一定长度就拆成两个）

最终生产流程：「选书 → 读书 → 提炼Skill → 发布GitHub → 生成效果示例 → 生成网站内容」
除"选书"外，所有步骤全部用Skill完成。

**6个Skill的架构**：
- 主Skill（book2skills-publisher）：编排整个流程，纯文字松散指令，AI可灵活判断
- read-book-skill：逐章提取PDF，整理成结构化markdown
- book-skill-creator：基于章节内容仿Anthropic规范提炼SKILL.md（质量影响最大的一步）
- write-skill-repo：自动生成GitHub文件夹+zip打包
- book-skills-examples：真实调用Skill，生成中英文使用示例（带真实数据）
- write-skills-page-content：整合成网站页面

**步骤三：写锚定文档（"What is book2skills"）**
- 给自己定边界
- 确保接下来的Skills不会过度设计，不会跑偏

**步骤四：持续促进Skill自我迭代**
- 用户发现问题，告诉AI"加这条规则"→ AI更新文件 → 下次立即生效
- 不需要任何开发者介入
- 迭代过程本身也可以自动化！

## 四、Skills的本质优势（为什么Harness Engineering能work）

1. **可并行迁移**：安装到任何环境直接用
2. **可自我迭代**：运行过程中积累reference和spec = 师傅给徒弟的"秘籍"
3. **AI自主更新**：人类不介入，AI自己查文档、自己更新规则

**"秘籍"写得好 → harness强 → 无人值守程度高**

---

## 五、Harness Engineering的局限性

1. Harness不难，但需要前期投入（2天构建期）
2. 不是一劳永逸，需要经常维护
3. Skills是前提——没有好的Skills，Harness搭不起来

## 六、对臣的启发（Harness Engineering × 多Agent协作）

臣的多Agent协作体系 = 内容生产领域的Harness Engineering：

| Harness Engineering（通用） | 臣的Agent体系 |
|--------------------------|--------------|
| "秘籍"（标准/原则/Skills/反馈循环） | Agent SOUL.md + 工作流SOP |
| AI自主查文档解决问题 | intel搜索Agent读SOUL.md执行 |
| 无人值守运行 | 情报官05:00自动触发，全程无需臣介入 |
| "秘籍"质量决定可靠程度 | SOUL.md质量决定Agent输出质量 |
| Skills自我迭代积累reference | 信念抽屉+洞察库持续沉淀 |
| 主Skill编排子Skill | edu_lead统筹edu_writer+5读者Agent |

**共同的核心逻辑**：
- 规则前置 + AI自主执行 + 反馈内化 = 高可靠输出
- 早期建立的框架（"秘籍"/"Harness"）决定后期行为质量

**与AI教学的深层连接**：
- 教学生用AI = 给他们一套认知Harness
- 早期植入，越早越好
- 这与陛下"AI伦理从第一堂课开始"的理念完全一致
