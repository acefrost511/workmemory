# 洞察写作规范（v4.0）
> 版本：v4.0 | 更新：2026-04-04 | 制定依据：陛下重构建议
> 定位：洞察生产线唯一执行标准（精简版，细节入skill）

---

## 核心原则

**触动、判断、成品——这三个节点必须由陛下本人完成，AI不能替代。**

---

## 日程表

| 时间 | 执行方 | 内容 |
|------|--------|------|
| 05:00 | 情报官 | 12路搜索，新文章入待审队列 |
| 05:40 | 简报skill | 生成10篇简报，推送陛下 |
| 06:30 | 系统 | 队列截止检查，dispatcher停止 |
| 收到「标记完毕」| collision-analysis skill | 碰撞分析，推送碰撞卡 |
| 收到判断 | insight-generate skill | 生成完整洞察+读者模拟反应 |
| 17:00 | insight-extend skill | 推送可扩展洞察Top3 |
| 18:00 | 系统 | 无回复则静默结束 |

---

## 流程接口（输入→输出）

### 接口1：简报 → 触动标记
- 输出：简报10篇写入 `/.daily_briefing.md`
- 飞书推送陛下，等「标记完毕」

### 接口2：触动标记 → collision-analysis skill
- 输入：陛下的触动文章路径列表 + 一句话触动理由
- 执行：读取触动原文 → 关联信念抽屉 → 搜索相关素材 → 生成碰撞卡
- 输出：碰撞卡列表 → 飞书推送陛下，等判断

### 接口3：判断 → insight-generate skill
- 输入：陛下对碰撞卡的判断（选方向 + 写判断原话）
- 执行：生成7部分洞察全文 → 并行spawn 4个读者Agent → 汇总模拟反应
- 输出：洞察全文 + 读者反应 → 飞书推送陛下，等审核

### 接口4：审核 → 存储
- 陛下回复：发布 / 改 / 不发
- 执行：写入INSIGHT_LIBRARY.md → 反哺对应信念抽屉

### 接口5：每日17:00 → insight-extend skill
- 输入：读取INSIGHT_LIBRARY.md
- 执行：筛选可扩展洞察（入库≥1天/未扩展/未否决/未推送3次）
- 输出：Top3推送陛下，等「用洞察#XX写文章」或「都不写」

---

## 关键限制条件

1. **队列截止：06:30** — 情报审核dispatcher每日06:30自动停止，队列清空与否均停止
2. **触动底线：3篇** — 陛下至少标记3篇触动，不足则不启动碰撞分析
3. **洞察编号：INSIGHT_NEW_XXX** — 新流程洞察与旧洞察编号不重叠
4. **读者Panel：4角色** — 新教师/资深教师/校长/K12教育专家，不打分
5. **反造假**：每篇引用必须有DOI，数字必须有来源，陛下原话一字不改
