_建立时间：2026-03-27 00:13_
_目标：从"已确认洞察"到"9分文章"的工业化流水线_

## 一、洞察确认机制

**触发条件：** 陛下通过飞书确认某条洞察（如"洞察1和洞察3很好，做文章"）

**确认后动作：**
1. 将该洞察标记为「创作态」（写入 `knowledge/creative_sparks.md` 对应洞察的元数据区）
2. 记录：确认时间 / 陛下点评（如果有）/ 优先级
3. 立即启动文章创作流程，不等待

---

## 二、文章创作标准模板

### 标题要求
- 主标题（≤20字，有冲击力，一击即中）
- 副标题（≤30字，补充说明）
- 标题公式：`反直觉Hook + 具体人群 + 悬念`

### 结构要求（5段式）
```
引入（Hook）：从具体场景/故事切入，200字
        ↓
问题揭示：为什么这件事值得关注，300字
        ↓
证据呈现：两条研究原文 + 通俗解读，500字
        ↓
核心洞察：作者（陛下/IP）对这个问题的深层理解，400字
        ↓
行动建议：给读者（老师/家长/学生）的具体做法，300字
```

### 语言标准
- 目标读者：一线教师、班主任、小学初中家长
- 句子长度：≤30字，避免长从句
- 专业术语：首次出现附中文解释（如"SAMR模型"→"SAMR模型，即替代→增强→修改→重新定义的四层教育技术整合框架"）
- 禁止：空洞金句、无数据支撑的断言、脱离场景的抽象论述

### 字数标准
- 初稿：1500-2000字
- 终稿：1800-2200字

## 三、多Agent评分迭代流程

### Agent配置
| Agent | 视角 | 评分维度 |
|-------|------|----------|
| 新教师Agent（1-3年教龄） | 一线教师实操 | 可操作性×3 / 可读性×2 |
| 成熟教师Agent（10年+） | 骨干教师专业 | 专业深度×3 / 逻辑严谨×2 |
| 校长Agent | 学校管理层 | 整体价值×3 / 可转化性×2 |
| K12教育专家Agent | 学术+政策 | 教育规律契合×3 / 创新性×2 |
| ~~新增：自媒体专家Agent~~ | ~~内容传播~~ | ~~读者吸引力×3 / 传播性×2~~ |

（新增Agent视迭代轮次决定是否引入）

### 评分标准（每条维度0-10分）
```
可操作性（教师/家长能否直接用）
可读性（文字是否通俗易懂、无障碍阅读）
专业深度（是否有理论支撑）
逻辑严谨（论证是否自洽）
整体价值（对读者是否有真实帮助）
```

### 迭代规则
- **第1轮**：生成初稿 → 4Agent并行评分 → 输出评分+修改建议
- **第2轮**：根据修改建议修改 → 4Agent并行评分
- **第3轮及以后**：继续迭代，直至综合分≥9.0
- **轮次上限**：5轮。5轮仍未达标，标记「需人工介入」，报告陛下
- **每轮输出**：
  - 各Agent评分（0-10分）及简短评语
  - 综合分（4Agent均分）
  - 最高优先级修改建议（≤3条）

### 达标标准
```
综合分 = (新教师分 + 成熟教师分 + 校长分 + 专家分) / 4
达标：综合分 ≥ 9.0
```

## 四、创作流程详细步骤

```
Step 1: 素材准备（Agent自动）
  - 读取已确认洞察的详细信息（来源血统+核心洞察+读者带走）
  - 读取对应抽屉的原始研究（≥2条）
  - 读取陛下笔记（如果有）

Step 2: 标题生成（5个候选标题）
  - 每个标题附带：主标题+副标题+取标题的理由

Step 3: 文章初稿（按结构模板）
  - 引入：从洞察的"Hook句"延伸，找一个具体场景
  - 问题：这条洞察为什么重要，对谁重要
  - 证据：引用原文，换成通俗说法
  - 洞察：陛下的核心观点（基于碰撞产出品）
  - 行动：给读者的3个具体做法

Step 4: 4Agent并行评分
  - 每个Agent输出：综合分(0-10) + 3条修改建议 + 1句最认可的部分

Step 5: 修改迭代
  - 汇总4Agent的修改建议，取最高优先级
  - 修改文章
  - 重复Step 4-5

Step 6: 终稿输出
  - 综合分≥9.0 → 标记「可用」，推送陛下
  - 5轮未达标 → 标记「需人工介入」，附上所有版本和评分记录

Step 7: 推送陛下
  - 终稿完整内容（不含评分数据）
  - 各版本分数变化趋势
  - 迭代总结（修改了什么，为什么改）
```

## 七、持续演进规则

1. **每篇新文章**必须建立版本历史，不可覆盖旧版本
2. **评分记录**永久保留，供后续创作参考
3. **优质表达库**：从每次迭代中提炼出高分句式，写入 `/workspace/knowledge/good_phrases.md`
4. **教训记录**：从每次失败迭代中提炼教训，写入 `/workspace/knowledge/iteration_lessons.md`
5. **Agent反馈积累**：定期汇总4个Agent最常提出的修改建议，优化创作prompt
