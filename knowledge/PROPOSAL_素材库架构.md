# 素材库架构方案（供陛下决策）

---

## 核心问题诊断

| 方案 | 优点 | 缺点 |
|------|------|------|
| 每篇单独文件（当前） | 简单 | 碎片化，IP团队无法判断读哪篇 |
| 按日期合并（臣建议） | 定时归拢、容易读 | 文档随时间变大 |
| 按主题合并 | 检索精准 | 主题边界模糊，归档判断复杂 |

---

## 臣建议的最终方案：双层文档结构

```
两层结构：
① 活跃素材库（持续写入、不断追加）
   └── IP团队从这里读取→归入②

② 13个信念抽屉（每个抽屉一份素材积累文档）
   └── IP团队归档后的最终存储位置
```

---

## 具体设计

### ① 活跃素材库（情报官维护）

**文件数量：按日期分段，适度合并**

方案A（每日一份）：
```
素材库/2026-03-27.md  ← 3月27日的所有新素材（情报官扫描+陛下笔记）
素材库/2026-03-28.md  ← 3月28日的所有新素材
素材库/2026-03-29.md  ← 3月29日的所有新素材
```

方案B（每积累20篇合并一份）：
```
素材库/活跃库.md      ← 持续追加，直到超过200KB自动拆分
                        拆分后归档为 历史库/2026-03-1.md
```

**臣建议：方案A（每日一份）**
- 理由：每日定时归档，IP团队节奏清晰
- 每次情报官扫描完 → 生成当日新素材文档
- IP团队每日读取当日文档 → 归入13个抽屉

---

### ② 13个信念抽屉（IP主编维护）

每个抽屉一份素材积累文档：
```
信念抽屉/信念1-AI教育观.md
信念抽屉/信念2-教师角色观.md
信念抽屉/信念3-学习本质观.md
...（共13个）
```

每份抽屉文档结构：
```
【信念X-XXX观】素材积累
更新时间：YYYY-MM-DD

---
【新素材1】（2026-03-27归档）
来源：素材库/2026-03-27.md
类型：研究论文 / 陛下笔记
核心观点：[一句话描述]
详细内容：
（完整粘贴研究概述或陛下素材原文）

---
【新素材2】（2026-03-27归档）
...
```

## 两种触发机制

### 机制一：实时触发（陛下发素材时）
```
陛下发送笔记/洞察/文章
    ↓
小艾接收 → 追加到当日活跃素材库
    ↓
小艾立即通知IP主编：「有陛下的新素材待归档」
    ↓
IP主编立即处理（实时归档到13个抽屉）
```

### 机制二：定时触发（每30分钟一次）
```
定时触发（小艾心跳或cron）
    ↓
检查活跃素材库是否有新的未归档内容
    ↓
有新增 → 通知IP主编处理
    ↓
无新增 → 不打扰
```

---

## 执行流程（完整闭环）

```
① 情报官扫描完成
    ↓
② 情报官生成当日素材文档（素材库/YYYY-MM-DD.md）
    ↓
③ 情报官通知IP主编：「今日新素材已入库，请归档」
    ↓
④ IP主编立即读取当日素材文档
    ↓
⑤ IP主编判断每条素材归属哪个信念抽屉
    ↓
⑥ 追加到对应抽屉的素材积累文档
    ↓
⑦ IP团队产出洞察（基于抽屉内容）
    ↓
⑧ 完成后通知小艾 → 小艾通知陛下
```

## 臣的问题（请陛下确认）

**问题1：活跃素材库按日期分段还是按积累量分段？**
- 方案A（每日一份）：清晰，但一个月有30个文件
- 方案B（每积累50篇合并一份）：文件少，但触发时机不固定

**问题2：IP主编是一个人Agent还是整个IP团队？**
- 如果是edu_lead一个人 → 负责归档+洞察，压力较大
- 如果是IP团队 → 可分工：有人负责归档，有人负责洞察

**问题3：活跃素材库是否有上限清理机制？**
- 超过7天的旧活跃库 → 转移至"历史库"存档（只读）
- 还是：每日活跃库归档后即清空，内容已在抽屉里？

---

## 臣的建议（供陛下参考）

- **活跃素材库：按日期每日一份**（最清晰）
- **历史库：超7天转历史**（减少干扰）
- **IP主编：整个IP团队**（edu_lead牵头，edu_writer/edu_reviewer协作）
- **实时+定时双机制**：陛下发素材实时触发 + 每30分钟定时检查

陛下看这个方向对不对？确认后臣立即执行落地。
