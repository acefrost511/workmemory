# IP团队多Agent工作流 v2.0
> 版本：v2.0 | 日期：2026-03-28 | 陛下确认版（新增动态权重）

## 架构原则

**两层架构：**
- 第一层：陛下 ←→ 小艾（唯一沟通接口）
- 第二层：IP主编 + edu_writer + 6个读者Agent（并行运作）

所有Agent由小艾预先创建，由小艾统一调度。
IP主编负责工作流协调，不创建子Agent。

---

## 团队角色清单（8个Agent）

| Agent | 角色 | 职责 |
|-------|------|------|
| edu_writer | 洞察撰写 | 基于信念抽屉写洞察初稿 |
| reader_new_teacher | 新手教师评分 | 1-3年教龄教师视角审核 |
| reader_senior_teacher | 成熟教师评分 | 5年以上教龄教师视角审核 |
| reader_principal | 校长视角评分 | 学校管理层/教研员视角审核 |
| reader_parent | 家长视角评分 | K12学生家长视角审核 |
| reader_expert | 教育专家评分 | 学术研究者/教研员视角审核 |
| edu_lead | 主编 | 工作流协调+终审+提交陛下 |

## 完整工作流（17:30定时触发）

```
17:30  小艾spawn edu_lead主编
           ↓
主编读取13个信念抽屉（最新素材）
           ↓
主编选择洞察主题（2-3个信念的碰撞点）
           ↓
主编spawn edu_writer写初稿
           ↓
Writer写出3个候选洞察
           ↓
主编并发spawn 6个读者Agent评分（同时）
           ↓
各读者Agent独立评分 + sessions_send回报主编
           ↓
主编汇总评分：
  ① 总分≥9.0 ② 无单维度≤4 ③ 社交货币≥6 ④ 传播力≥6
           ↓
  ├─ 全部达标 → 主编把关 → 飞书推送陛下
  │
  ├─ 未达标 → 打回Writer（告知修改意见）→ 迭代
  │      ↑________________________|
  │         （最多3次迭代）
  │
  └─ 3次迭代后仍<9.0 → 放弃该洞察，换候选
           ↓
主编推送陛下：今日洞察结果（通过/放弃）

并行：每篇洞察完成 → 主编立即归档反哺抽屉
```

## edu_writer（洞察撰写）

**输入：**
- 读取2-3个信念抽屉（核心表述+金句+素材积累区）
- 从素材库提取相关研究

**输出：**
- 3个候选洞察（每篇含7部分：Hook≤60字/研究引用/核心洞察350-450字/读者带走老师≥3条/读者带走家长≥2条/DOI/虚构声明）

**写作要求（v5.0）：**
- Hook ≤60字，必须有数字或反常识
- 核心洞察350-450字
- 读者带走·老师≥3条（每条≥30字）
- 读者带走·家长≥2条（每条≥30字）
- 每篇洞察的文献引用必须：英文标题（中文翻译在括号内）+ 作者 + 机构 + 期刊 + 发表年月 + DOI
- 禁止统计量（p值/z值/F值等），用"研究证实"/"两组差异明显"替代
- 禁止虚构场景/人物

## 6个读者Agent（并发评分）

每个读者Agent同时独立执行，互不等待。

### 评分维度分工

| Agent | 核心审核维度 | 通过线 |
|-------|------------|--------|
| reader_new_teacher | Hook吸引力+情感共鸣+读者带走 | ≥7.0 |
| reader_senior_teacher | 教学落地性+认知增量+实操可行性 | ≥7.0 |
| reader_principal | 准确性+知识增量+公平性 | ≥7.0 |
| reader_parent | Hook吸引力+情感共鸣+节奏可读性 | ≥7.0 |
| reader_expert | 学术严谨性+理论深度+实践相关性 | ≥7.0 |

### 动态权重机制（主编根据洞察类型调整）

**主编在分配评分任务时，必须先判断洞察类型，再确定权重。**

#### 洞察类型分类标准

| 类型 | 判断标准 | 典型话题 |
|------|---------|---------|
| 教师向 | 核心受众是教师，讲教师课堂实践/教学方法/AI工具使用 | AI批改作业/AI备课/AI课堂互动/教师专业发展 |
| 家长向 | 核心受众是家长，讲家庭场景/子女教育焦虑/亲子沟通 | AI陪伴学习/AI作业辅导/屏幕时间/家长监督 |
| 校长向 | 核心受众是学校管理层，讲政策落地/教师培训/学校规划 | AI教师培训/学校AI部署/教研管理/教育公平 |
| 学术向 | 有较强理论框架，研究数据丰富，适合学术读者 | 系统性综述/理论模型/国际比较研究 |
| 通用向 | 话题横跨多个群体，无明确单一受众 | AI教育整体趋势/跨场景洞察 |

#### 权重调整规则

**基准权重（通用向）：**
- 读者合计权重：40%
- 专家权重：15%

**教师向洞察（教师权重提高）：**
- reader_new_teacher × 1.5
- reader_senior_teacher × 1.5
- reader_parent × 0.6
- reader_principal × 0.8
- reader_expert × 0.8
- 最终归一化处理

**家长向洞察（家长权重提高）：**
- reader_parent × 1.5
- reader_new_teacher × 1.0
- reader_senior_teacher × 0.7
- reader_principal × 0.6
- reader_expert × 0.7
- 最终归一化处理

**校长向洞察（校长+专家权重提高）：**
- reader_principal × 1.5
- reader_expert × 1.3
- reader_parent × 0.5
- reader_new_teacher × 0.6
- reader_senior_teacher × 0.8
- 最终归一化处理

**学术向洞察（专家权重提高）：**
- reader_expert × 1.8
- reader_principal × 1.2
- reader_parent × 0.5
- reader_new_teacher × 0.6
- reader_senior_teacher × 0.8
- 最终归一化处理

#### 主编判断流程

**Step 1：主编读完洞察草案后，先判断类型**
- 看Hook和核心洞察：谁最可能打开这篇文章？
- 看读者带走：哪类读者得到的行动建议最具体？

**Step 2：确定类型后，告知所有读者Agent调整后的权重**
- 主编在spawn读者Agent时，在prompt中注明："本次洞察类型：[类型]，你的权重为：×[N]"

**Step 3：汇总时使用调整后的权重计算**
- 不同洞察的总分不直接比较（因为权重不同）
- 每一篇洞察内部，权重调整后的总分才是有效分数

**通过判定（必须同时满足）：**
① 调整后加权总分 ≥ 9.0
② 无任何单维度平均分 ≤ 4
③ 目标读者维度 ≥ 6（教师向洞察：reader_new_teacher+senior平均≥6；家长向：reader_parent≥6）
④ 社交货币指数 ≥ 6
⑤ 文章传播力 ≥ 6

### 评分标准（8维度加权）

```
① 学术严谨性 权重15%：DOI可查证/无统计量虚构
② 内容新颖性 权重15%：视角独特/有认知增量
③ 实操落地性 权重20%：老师可直接落地/家长有具体做法
④ Hook钩子力 权重20%：标题吸引/开头抓住注意力
⑤ 社交货币指数 权重15%：愿意转发/是社交谈资/"原来如此"感叹
⑥ 文章传播力 权重15%：打开率/完读率/收藏率/转发率潜力

淘汰红线：
- 社交货币指数<6 → 直接否决（不值得生产）
- 文章传播力<6 → 直接否决（不值得发布）
- 任一单维度≤4 → 直接否决
```

## edu_lead主编（工作流协调+终审）

**主编职责：**
1. 读取信念抽屉，选择洞察主题
2. spawn edu_writer写初稿
3. 并发spawn 6个读者Agent评分
4. 汇总评分，判断是否达标
5. 处理迭代（打回Writer重写）
6. 终审把关（≥9.0）
7. 飞书推送陛下
8. 反哺抽屉（洞察结果写回信念文件）
9. 永久存储（/knowledge/insight_daily_YYYY-MM-DD.md）

**主编汇报陛下格式（洞察完成后）：**
```
━━━━━━━━━━━━━━━━━━
📝 今日洞察 | YYYY-MM-DD

洞察编号：洞察X
信念溯源：信念{N} × 信念{N}
总分：[X/10] | 社交货币：[X] | 传播力：[X]

评分明细：
reader_new_teacher：[X/10] ✅/❌
reader_senior_teacher：[X/10] ✅/❌
reader_principal：[X/10] ✅/❌
reader_parent：[X/10] ✅/❌
reader_expert：[X/10] ✅/❌

迭代次数：X次（≤3次）

结论：[通过发布 / 放弃（3次未达标）]

洞察全文：[完整内容]

附：洞察永久存储路径
━━━━━━━━━━━━━━━━━━
```

## 迭代规则

- 第1次迭代：Writer写初稿 → 6读者评分 → 汇总<9.0 → 打回修改
- 第2次迭代：Writer修改 → 6读者重评分 → 汇总<9.0 → 再打回
- 第3次迭代：Writer第2次修改 → 6读者第2次评分 → 汇总<9.0 → 放弃该洞察，换候选

**每次打回时，主编必须告知Writer：**
- 哪些维度被扣分（具体到哪条）
- 读者A/B/C的具体意见（引用原文）
- 重点修改方向（≤3条）

## 反哺抽屉规则

洞察通过后，主编执行（用exec/bash，不依赖edit）：
```bash
cat >> /workspace/knowledge/beliefs/信念{N}-{名}.md << 'EOF'

### [洞察-YYYY-MM-DD-洞察X]
- 来源：信念{N} × [研究A] × [研究B]
- 新认识：[2-3句话]
- Hook：[Hook句]
- 全文：/workspace/knowledge/insight_daily_YYYY-MM-DD.md
EOF
```

## 搜索质量红线（情报官执行）

**入库必须同时满足：**
1. ✅ 有DOI（或arXiv编号/机构官方链接）
2. ✅ 研究对象是K12阶段
3. ✅ 主题含"AI教育"或"AI教学"关键词

**禁止入库：**
- ❌ 高等/成人继续教育AI
- ❌ 师范生AI学习（无K12课堂关联）
- ❌ AI伦理/AI安全（无K12教学场景）
- ❌ 情感计算/纯心理学（无教育维度）
- ❌ 单独"AI"或"education"（无交叉）
