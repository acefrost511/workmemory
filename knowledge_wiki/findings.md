# 第一部分：笔记库与主题精选库融合方案

## 一、核心融合逻辑

笔记库与主题精选库不是两个独立仓库，而是同一条内容生产线的不同阶段：

```
随手记（raw notes）
    ↓ 清洗+提炼（臣执行）
结构化笔记（processed notes）
    ↓ 知识图谱关联（臣执行）
笔记簇群（topic bundles）= 主题候选
    ↓ 陛下审核精选
主题精选库（prioritized topics）= 待写作选题
    ↓ 臣执行写作
公众号文章（published content）
    ↓ md2wechat排版
微信公众号发布
```

## 二、融合触发条件

当以下条件满足时，臣主动向陛下建议将笔记簇升级为精选主题：

1. 同主题笔记≥3条
2. 笔记quality_score均值≥3.5
3. 笔记之间形成≥1个可量化观点/数据
4. 至少有1个具体K12教学场景

## 三、笔记→文章的写作流程

```
陛下选定主题（来自精选库）
    ↓
臣检索该主题相关所有笔记（tags + related_topics）
    ↓
笔记排序：quality_score高→低，时间新→旧
    ↓
笔记内容提炼 → 形成文章核心观点簇（3-5个）
    ↓
辅以素材库文献（补充数据+来源背书）
    ↓
调用edu-editor写作
    ↓
输出草稿 → 陛下审核
    ↓
md2wechat排版 → 发布
```

## 四、笔记的"Dan Koe式"价值倍增

借鉴Dan Koe的方法论，笔记的价值在以下环节被放大：

**笔记即选题验证**：陛下发来某个方向的碎片想法 → 臣整理成笔记 → 通过知识图谱看是否已有相关积累 → 有积累则说明这个方向值得深挖 = 选题已验证

**笔记即内容积木**：每条笔记是一个"想法积木"，写文章时从笔记库抽取积木，组合成完整文章，而非从0开始写

**笔记即分发原点**：一篇公众号文章 = 1篇Newsletter = N条朋友圈/群发内容，笔记是所有分发内容的原始积木

# 第二部分：Dan Koe内容创作系统 · 完整方法论

## 一、核心框架：一次创作，多点分发

Dan Koe不追逐灵感，他的系统让创意被验证后再大投入：

**创意来源（已验证）**：
- 表现最好的Twitter帖子 → 扩展成Newsletter
- 所在领域热门YouTube视频主题 → 转化为自己的观点

**分发路径**：
```
Twitter帖子（280字=最强约束，迫使浓缩精华）
    ↓ 验证（数据好=继续）
Newsletter（1000-2000字=完整展开）
    ↓ 脚本
YouTube视频（5-10分钟=深度演绎）
    ↓ 拆解
新的Twitter帖子（从视频核心观点反向生成）
```

**核心原则**：宁可每周只做一件惊艳的作品，把全部注意力放进去，再分发到各平台。

## 二、AI的正确使用方式

**AI是"实习生"，不是"代笔"**：

1. **信息浓缩**：用Gemini/NotebookLM把6小时视频压缩成1000字摘要
2. **交叉联想**：把AI摘要 + 自己的旧笔记一起喂给AI，问："这二者之间有什么相似之处？我遗漏了什么？"
3. **爆款逆向工程**：找到3条高表现帖子 → 丢给AI → AI输出结构模板 → 生成新内容框架
4. **元提示词**：让AI学会像"你"一样思考，再执行任务（两阶段提示词系统）

**对陛下适用的版本**：
- 音频/视频学习 → AI浓缩摘要 → 臣提炼核心观点 → 存入笔记库
- 笔记群 → AI分析关联 → 形成文章框架

## 三、创作者核心优势：分离"想法"与"结构"

Dan Koe的Swipe File（灵感收藏库）= 陛下的"笔记库"

两种练习：
- 练习A：选一个内容框架模板，插入一个新想法
- 练习B：选一个想表达的想法，用不同结构各写一遍

**任何想法，无论多简单，都能被装进经过验证的传播框架。**

## 四、一人一商业模式（OPB Model）

Dan Koe变现四步：

1. **MVO（最低可行报价）**：单兴趣咨询套餐，$500-$1000
2. **产品化**：将咨询内容转化为数字产品（课程/模板/工作表）
3. **升级**：产品买家转化为高价客户
4. **规模化**：建立受众后推出新产品/服务

**核心洞察**：你的第一个产品会很烂，立即推出再迭代比等待完美更有效。

**你的独特优势**：

- AI工具一手使用经验（不是理论派）
- 研究论文解读能力（把学术转化为实操）

## 二、内容选题策略

**来源一（验证过）**：情报官推送的高质量研究 → 解读写成文章

**来源二（验证过）**：陛下笔记库中的高频主题 → 整理成文

**来源三（自发）**：K12教学中的真实痛点 → 转化为解决方案文章

**选题原则**：
- 写帮助教师解决实际问题的内容
- 每个选题必须有1个可量化数据或真实案例
- 标题/开头必须用具体场景，不用"浅谈""浅析"

# 第四部分：笔记库完整操作手册

## 入口协议（陛下须知）

**发语音**：直接发语音/录音，臣自动处理全流程，5分钟内完成

**发文字**：粘贴文字片段，臣判断并清洗，不需要陛下整理格式

**发链接**：发文章/论文链接，臣提炼核心观点存为笔记

**发感慨**：随口说的想法，臣提取有价值内核，存入笔记库

## 臣的承诺

- 陛下发来内容 → 立即开始处理，不等待
- 清洗后的笔记 → 标注summary，陛下可查阅
- 不确定的口误 → 直接括注英文原文（不写"待确认"）
- 笔记达到写作标准 → 主动建议陛下："X主题已积累成熟，建议优先写"

1. 读取近7日所有笔记
2. 读取笔记索引和主题库状态
3. 生成300-600字教育洞察推送陛下
4. 标注高质量笔记和建议写作主题
