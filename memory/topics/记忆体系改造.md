## Claude Code三层记忆架构

1. **工作记忆**：conversation_buffer.md（实时追加每轮对话）
2. **会话记忆**：memory/daily/YYYY-MM-DD.md（按日归档）
3. **持久记忆**：MEMORY.md（纯索引≤200行）+ topics/*.md（按主题）

## 核心原则

- MEMORY.md 是纯索引，不含实质性内容
- 索引行≤150字符
- topics/ 是实质性内容的家，不是索引
- 按主题组织，不按时间

## 两步保存法

每次记忆写入：先写文件 → 再更新索引（不能只写索引不写内容）

## 独立记忆抽取 Agent

主Agent不自己写记忆，由独立子Agent负责
cron：`9d91812f`（每5分钟自动触发）
职责：读conversation_buffer → 提取有价值内容 → 写入topics + 更新索引

## 做梦机制

每日凌晨自动整理记忆（去重/修正/更新索引）
cron：`bb6e8619`（每日04:00）
四阶段：盘点现状→收集信号→整合修正→修剪索引
输出：发现X个问题，已修正Y个，MEMORY.md当前Z行
