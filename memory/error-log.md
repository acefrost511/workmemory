## 2026-03-05 - 系统 - 网络代理配置修复（关键修复）

### 问题描述
web_fetch工具和子Agent无法访问国外网站，但用户电脑可以通过Clash正常翻墙。

### 原因分析
1. 环境变量未设置HTTP_PROXY/HTTPS_PROXY
2. OpenClaw Gateway未配置代理
3. 子Agent未继承代理环境变量

### 修复过程
1. **检测代理配置**：发现系统WiFi代理设置为 127.0.0.1:7897（Clash）
2. **配置环境变量**：添加到 ~/.zshrc
   - HTTP_PROXY=http://127.0.0.1:7897
   - HTTPS_PROXY=http://127.0.0.1:7897
   - ALL_PROXY=http://127.0.0.1:7897
3. **更新easyclaw.json**：在agents.defaults.env中添加代理配置
4. **验证连接**：curl通过代理成功访问Google

### 最终修复（彻底解决）

**根本原因**：OpenClaw的exec工具启动的是non-login shell，不读取 `~/.zshrc`，导致代理环境变量没有生效。

**解决方案**：写入到三个层级，确保所有环境都能读到：
1. `~/.zshenv` — 所有zsh进程都读（包括non-login shell）✅
2. `~/.profile` — bash/sh环境 ✅
3. `launchctl setenv` — macOS全局环境变量，GUI应用也生效 ✅

**验证结果**：
- EdSurge ✅ HTTP 200
- UNESCO ✅ HTTP 200
- Google ✅ HTTP 200

### 修复文件
- ~/.zshrc - 添加代理环境变量
- easyclaw.json - 添加agents.defaults.env代理配置
- PROXY-CONFIG.md - 代理配置文档

## 2026-03-05 - 系统 - 学习实战文章后补充修复（P0级）

### 来源
学习了《Team Setup v2.0》和《从翻车到飞轮》两篇实战文章，发现4个之前遗漏的技术问题。

### 问题1：子Agent runTimeoutSeconds未配置 【已修复】
- **问题**：spawn子Agent没有设置超时，系统默认值可能很短，复杂任务被强制kill
- **后果**：百晓生的搜索Agent运行到一半被kill，留下不完整状态
- **修复**：`easyclaw.json` 中设置 `subagents.runTimeoutSeconds = 2400`（40分钟）
- **超时参考**：简单任务1200s，中等任务1800s，复杂任务2400s（最少不低于1200s）
- **kill后必做**：检查残留的partial文件，记录到错题本，用其余Agent数据继续

### 问题2：10个搜索Agent同时启动触发Rate Limit【已修复】
- **问题**：10个Agent同时spawn，大量并发API请求触发限流
- **修复**：分两批启动，第一批5个（05:00），第二批5个（05:05），错开5分钟
- **原则**：并发控制 > 什么都不做，宁可慢5分钟，不要因为限流全部失败

### 问题3：记忆膨胀，上下文装不下【已修复】
- **问题**：每天一个memory日志，时间长了Agent读取历史时上下文爆炸
- **修复**：建立三层记忆体系（原始日志 → 周摘要 → 长期精华+INDEX索引）
- **参考**：shared-skills/task-scheduling/memory-curator.md

### 问题4：Workspace备份无验证机制【记录，后续跟进】
- **问题**：只有GitHub同步，但没有验证备份是否完整、是否可以正常恢复
- **教训**：重装系统后AI完全失忆，workspace被清空，从头开始
- **后续**：每周验证一次GitHub备份是否完整，检查关键文件（SOUL.md、MEMORY.md等）
- **恢复注意**：恢复后必须删除BOOTSTRAP.md，否则AI会重新初始化覆盖已有配置

### 已确认正常的配置
- `compaction.mode = safeguard` ✅（防tool_use_id不匹配报错）
- `sessionTarget: isolated` ✅（各cron任务独立session，不会无限增长）
- `subagents.maxConcurrent = 8` ✅（合理的并发上限）
