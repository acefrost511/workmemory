## 日报编号与审核规则

### 编号规则
- **全局顺序编号**：每日编号按顺序往下延续，不要每次新产品或新报告又从1开始编号

### 审核列表格式（必须包含具体内容）
日报审核列表必须包含以下字段，按三个部分分类（新研究、新报告、新产品）：
1. **序号** - 全局顺序编号
2. **资讯标题** - 吸引点击的标题（英文可保留加中文翻译）
3. **发表时间** - 具体发布时间
4. **内容简介** - 300-400字，详细描述内容，支持审核
5. **来源机构** - 发布机构/期刊/媒体
6. **建议优先级** - 高/中/低
7. **是否建议深度解读** - 是/否

## 今日教训与指导（2026-03-03）

### 犯过的错误
1. **任务执行失败**：cron任务配置了whatsapp频道导致失败，未检查上下文就执行
2. **遗漏历史内容**：之前已推送的6条资讯在最终版里丢失，没有检查之前的内容
3. **格式精简过度**：初次给出的审核列表只有简化版（#1-#7，#8-#13），缺少关键字段
4. **任务超时后被动等待**：日报任务超时后没有主动拆分任务重试

### 用户指导（必须记住）
1. **任务失败时的处理**：
   - 不要等用户提醒，要主动想各种办法解决
   - 可以拓展agent框架或拆分任务分步骤进行
   - 拆分后并行执行比串行更快

2. **审核列表格式（7个字段，缺一不可）**：
   - 序号
   - 资讯标题（英文可保留+中文翻译）
   - 发表时间 ← 关键
   - 内容简介（300-400字）← 关键
   - 来源机构
   - 建议优先级（高/中/低）
   - 是否建议深度解读（是/否）

3. **分类要求**：必须按三个部分分类（新研究、新报告、新产品）

4. **内容完整性**：每次推送前必须检查之前已推送的内容，确保不遗漏

5. **推送频道**：配置cron时delivery mode设为none，避免频道错误

## 今日教训（2026-03-04）

### 犯过的错误
1. **内容简介字数不足**：推给用户的审核列表中内容简介只有100字左右，没有达到规定的300-400字
2. **格式错误**：使用了表格格式，而不是用户要求的清单方式
3. **未检查就推送**：子agent生成后没有检查字数和格式就直接推送

### 教训总结
- 推送前必须检查：内容简介是否达到300-400字
- 格式必须是清单方式，不能用表格
- 所有的输出都要考虑人能不能理解，不能自己看懂就行
- 严格按MEMORY中的规定执行，不能偷懒
