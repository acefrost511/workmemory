## 工作原则（2026-03-05 确立，2026-03-06 追加）

**事事有回音，件件有回应**（王老师核心要求）

### 通知≠等待
- **通知是让用户知道进展，不是等用户回应**
- 通知完后**继续推进工作**，不要停下
- 每个阶段完成后：通知 → 立即进入下一阶段

### 具体执行
1. **阶段完成** → 通知王老师 → **立即继续下一步**
2. **遇到问题** → 通知王老师 + 同时想办法解决 → **不等用户指示**
3. **任务完成** → 通知王老师 → 等待新任务

### 记住
- 用户不需要操心过程，只需要知道进展
- 通知是"汇报"，不是"请示"
- 只有审核环节才需要等王老师回复

**任务成功率标准（2026-03-06 王老师明确要求）**
- **所有任务必须100%成功**
- 如果达不到100%，必须自己想办法解决
- 必须有多重备用机制（至少3层）
- 必须有本地缓存兜底
- 必须有失败监控和自动恢复
- 必须预演验证才能上线
- **绝不接受低于100%的成功率**

**失败问题解决机制（2026-03-06 王老师指示）**
- 遇到失败或超时问题，**不轻易放弃**
- **第一步**：小艾推动百晓生、CTO协作分析原因
- **第二步**：去GitHub、技术社区学习类似问题的解决办法
- **第三步**：学以致用，直到问题攻克
- **记住**：用户电脑能访问外网，说明网络没问题，要找其他原因

## 日报编号与审核规则

### 编号规则

### 审核列表格式（必须包含具体内容）

1. **序号** - 全局顺序编号
2. **资讯标题** - 吸引点击的标题（英文可保留加中文翻译）
3. **发表时间** - 具体发布时间
4. **内容简介** - 300-400字，详细描述内容，支持审核
5. **来源机构** - 发布机构/期刊/媒体
6. **建议优先级** - 高/中/低（附100字理由）
7. **是否建议深度解读** - 是/否（附100字理由）
