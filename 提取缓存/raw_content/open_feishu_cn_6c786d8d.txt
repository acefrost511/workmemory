# Get started quickly
[For more detailed development manual instructions, please directly visit Feishu Knowledge Base](https://bytedance.larkoffice.com/wiki/CZ0Aw2kPwiNLRxkfqoOc6IFjnne)
## Create widget
### Install developer tools
If the runtime console displays a get lark session error, please check whether the globally installed opdev version matches the dependency version in the project's package.json:
1. opdev >=3.3.0 matches @lark-opdev/block-docs-addon-webpack-utils >=1.0.0
2. opdev <3.3.0 matches @lark-opdev/block-docs-addon-webpack-utils <=0.0.5
> \* If you have installed the developer tools, please execute `npm uninstall @bdeefe/opdev-cli -g`to clean up the environment first, and then use the installation package to install it to avoid overwriting the path.
> \* Installation using the command line requires Node.js . Please refer to [Node.js for](https://nodejs.org/) the installation method. When installing Node.js, npm will be automatically installed.
> \* Opdev For more command operations, see: [Feishu Developer Tools - Command Line - Development Document - Feishu Open Platform](/document/no\_class/feishu-developer-tool---command-line#59e404d7)
```
npm install @lark-opdev/cli@latest -g
npm uninstall @bdeefe/opdev-cli -g
# Verify that the installation was successful
opdev help
```
### Log in to your account
1. After the installation is successful, execute the following command and follow the prompts to log in on the login page that opens:
```
$ opdev login
```
1. Select Development Environment, here select Feishu Development Environment![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/16e8c8eb41aa74233742d45d71d95eab\_DWMBmdC2Dv.png?height=320&lazyload=true&width=1952)
2. After selecting Development Environment, the browser will automatically start to jump to the login page to log in. After the login is successful, execute `opdev login`again and you will see the following output:![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/62266c6f30a418b379f46881ed44bfc6\_2UuLi71nXx.png?height=350&lazyload=true&width=2634)
### Creating a project
Execute the following command at the command line end point to create the sample project:
```
$ opdev create [FolderName]
// Example
$ opdev create demo
```
### Select component type
Select docs-addon here![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/0d738c3fb4b3a68ad5d608598615c3ef\_LBoRJiqqGF.png?height=244&lazyload=true&width=1686)
### Create a cloud doc block
Enter the Enter key, and the cloud document widget will be created automatically. You can click the link of the console to access the developer console to view the information about the cloud document widget created by the system.![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/2b97c64dedadb99b08d90fed2279ae09\_6xy7KbtTqz.png?height=166&lazyload=true&width=1280)
### Sample block introduction
This block allows you to insert the Mermaid charting tool into documents, enabling you to create complex diagrams such as flowcharts and sequence diagrams with a few lines of text (code).![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/09edcdcb1ffd286fdf25f9ea8b843429\_sa6jzhZ4zy.png?height=684&lazyload=true&width=1770)
```
├── README.md
├── app.json
├── node\_modules
├── package.json
├── src
├── tsconfig.json
└── webpack.config.js
```
## Debug block
Enter the project root directory, install dependencies, and start the project:
```
cd demo
npm i
npm start
```
In the open test page , you can insert the sample block through the menu (the local block name is blockTypeID)![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/0558195ec93fd1f711a92133d6ebdca5\_VK4fdOm095.png?height=808&lazyload=true&width=1104)
## Upload block
Execute the following command in the command line end point, and follow the prompts to complete the release of the applet.
```
npm run upload
```
## Publish block
1. Then in the app release column, create a version release![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/85eafb3b97a850cf445aabbd7e21d8d0\_QMhfYVUF4U.png?height=1340&lazyload=true&width=3486)
2. After the version is created, apply for online release, and then review by the![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/628c9dc4d3ddea63c57c8e3c2188ea4e\_XbSjxS29rr.png?height=902&lazyload=true&width=2956)background administrator. After the review is passed, the release is complete.
## Block Internationalization Configuration
Block Internationalization configuration allows you to display the corresponding block name and description in different locales after your block is published.To enable the Internationalization configuration, go to Open Platform - Credentials and Basic Information - Internationalization Configuration and add different language names and app descriptions.![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/011f1039f4d234763d13410e31df78cf\_OndQAGnqfR.png?height=1236&lazyload=true&width=2040)Then, add the basic information configuration of the block in different languages to your block and publish it to complete the block Internationalization configuration.![](//sf3-cn.feishucdn.com/obj/open-platform-opendoc/6d88efd122f2a33a927a373a96f0c5b2\_eZ3qQM0qrT.png?height=1390&lazyload=true&width=2286)In-App Internationalization can combine your technology selection and choose the right solution (such as [react-i18next](https://react.i18next.com/) )
We provide the corresponding interface to get the language of the current user:[Env.Language.getLanguage](/document/uAjLw4CM/uYjL24iN/docs-add-on/05-api-doc/basic-data-reference---base/Env.Language.getLanguage)
## Block configuration instructions
### Project configuration
The project configuration is located in `app.json`under the project root directory, which mainly contains the following configuration :
| Field name | Type | Description | Example |
| --- | --- | --- | --- |
| manifestVersion | number | Manifest version | 1 |
| appID | string | App ID, obtained from Feishu Open Platform | cli\\_a33d20ce6eb8\\*\\*\\*\\* |
| blockTypeID | string | blockTypeID, obtained after the block capability is enabled in the Feishu Open Platform | blk\\_63b79cd3cf80000150d6\\*\\*\\*\\* |
| contributes | Record | Component Type & Affiliate View Configuration | {"addPanel": {"initialHeight": 46,"view": "index.html"}} |
| projectName | string | Block name | Docsaddon-Demo |
For more configuration , please refer to [App.json configuration instructions](/document/uAjLw4CM/uYjL24iN/docs-add-on/03-cloud-doc-block-rapid-development-guide/appjson-configuration-instructions).

## illustrate
### Who will be affected?
The User Access Token obtained by the cloud document widget will not be able to call OpenAPI starting from January 22, 2024. \\*\\* If you need to use User Access Token to call OpenAPI, the business party needs to make relevant adaptations to ensure that product functions are not affected.
\* Use [Cloud Document Widget to obtain User Access Token](/document/uAjLw4CM/uYjL24iN/docs-add-on/05-api-doc/basic-data-reference---base/Service.User.login), and use the Token to call the OpenAPI application
### Upgrade Guide
Determine if you are affected? Do I need to use User Access Token to call OpenAPI?
\* No, the business does not need to use User Access Token to call OpenAPI, so it will not be affected.
\* Yes, if the business needs to use User Access Token to call OpenAPI, then adjust the product logic to obtain a User Access Token that supports calling OpenAPI.
### How to adjust product logic?
\*\*Option 1:\*\* Adjust the application authorization logic to jump to the third-party page within the widget and complete the authorization on the third-party page. You can refer to [Development Documents - Feishu Open Platform](/document/uAjLw4CM/ukTMukTMukTM/reference/authen-v1/login-overview) to add jump capabilities in the widget to jump to the third-party page to guide user authorization Related functions will be provided later.\*\*Option 2:\*\*[Use the Mini Program API](/document/uYjL24iN/ukjM04SOyQjL5IDN) to obtain openID, and use tenant access token to call OpenAPI to perform other operations, please refer to [Example](/document/uAjLw4CM/uYjL24iN/docs-add-on/05-api-doc/basic-data-reference---base/Service.User.login).
