《同事.skill 爆火后，我开始想：老师的日常工作哪些值得做成 skill？》

上周五，我的各种AI群里被一个 GitHub 项目刷屏了——"同事.skill"。把离职同事的聊天记录喂给 AI，三天炼出一个能 24 小时干活的数字分身。代码规范、甩锅姿势、技术判断风格，原汁原味。

不过"skill"这个词，你可能不完全熟悉。简单说：大模型相当于一个刚建号的满级游戏角色，自身能力很强，但面对具体任务时，它需要装备不同的"技能包"才能发挥最大战力。 skill 就是这样一个标准化的技能包——里面装着做这件事的完整知识、流程和质量标准。

同事走了，他的代码还能跑。那老师呢？

教了 30 年书的老教师退休了，他的教法谁来传承？——这个问题值得认真想一想。但在那之前，还有另一个更基础的问题：

你知道老师日常做的哪些工作，其实最值得做成 skill 吗？

一、教学 skill 和教案有什么区别？

在说"哪些"之前，先说清楚"是什么"。

很多老师听过"数字化教学资源""教学资源库""精品课程"这些词，以为把教案电子化就是做了 digitize 工作。但 skill 和教案是两件本质不同的事。

教案是一份静态文档。 它记录了"这节课我打算怎么做"，但它不会判断你的设计是否合理，不会针对这个班的学情提出调整建议，也不会在你备下一节课时主动调用上一次的经验。

skill 是一个可执行的系统。 它有明确的输入、有判断逻辑、有输出标准。你告诉它"高一数学三角函数诱导公式新授课"，它能生成一节完整的教学设计。你把一节课的教案丢给它，它能以教研员、学优生、学困生三个视角同时审阅，分别给出修改意见。

用一个公式表达：

教案 = 静态文档

skill = 输入 × 判断逻辑 × 输出标准

skill 不是教案的升级版，而是另一类工具——教案用来记录，skill 用来执行。

教学场景下，skill 有三层价值：

  • 省时间：常规环节（命题分析、学情统计、作业批改记录）可以交给 AI 处理
  • 保底线：让年轻老师的教学设计不低于学科平均水准
  • 助新手：新教师入职第一年就能拿到教研组积累的最佳实践，而不是从零摸索

二、老师的工作，哪些值得做成 skill？

不是所有教学环节都适合 skill 化。用 nuwa-skill 的一个核心概念来说，首先要区分行为层和思维层：

行为层（操作流程） 思维层（认知框架）
特点 有明确输入输出，可标准化 需要判断和决策，有情境依赖
skill 化难度 低，容易直接封装 高，需要提炼判断逻辑
适合人群 一线教师，直接拿来做 专家型教师，负责提炼

第一篇面向一线教师，优先解决行为层的任务。 思维层的内容留给第二篇。

下面是一张按教学流程整理的任务全景图——每个任务都经过一个简单判断：这件事有没有相对固定的输入和输出？有没有可复用的最佳实践？

如果是，做成 skill 的价值就高。

三、四个核心 skill 详解

上面每项任务都可以单独做成 skill。但 skill 的制作需要投入时间精力，优先级最高的是那些高频、标准化程度高、对教学质量影响大的任务。

以下四个 skill 同时满足这三个条件，是教研组最值得优先制作的。

Skill 1：lesson-designer（新授课设计）

一句话说清楚： 输入学科/年级/单元/课时，输出符合教研组标准的一节完整教案。

输入：

学科：高中数学
年级：高一
单元：三角函数
课时：诱导公式（第一课时）
课时目标：学生能运用诱导公式求任意角的三角函数值
班级学情：中等水平，运算能力薄弱，前置知识"单位圆与象限角"掌握率约70%

输出（示例片段）：

【导入环节】（建议时长：5-7分钟）
目标：激活单位圆知识，为诱导公式引入做铺垫
操作：
① 黑板画单位圆，快速点名回答四个象限的坐标特征（2分钟）
② 追问："如果把角写成 (π/2+α)，点在单位圆上的位置会怎么变？" 引出本节核心问题（2分钟）
③ 板书课题，说明本节要解决的核心问题：如何把任意角的三角函数值转化为锐角的三角函数值（1分钟）

[后续：新知建构→例题→变式→总结 完整输出]

提示词模板：

请帮我设计一节新授课。

- 单元/章节：____
- 课时主题：____
- 课时目标（须用"学生能……"的句式）：____
- 班级学情（可选）：____

## 输出格式要求
请输出一份完整教案，包含以下环节：
① 导入：激活前置知识的活动设计，时长建议
② 新知建构：概念引入路径，关键追问设计
③ 例题讲解：例题选择理由+讲解步骤+等待时间设计
④ 变式练习：至少3道变式，说明变式间的梯度设计逻辑

- 课时总长默认40分钟，如需调整请说明触发条件

Skill 2：teacher-lesson-reviewer（教案预审）

一句话说清楚： 把一节课的教案丢进去，模拟教研员、学优生、学困生三个视角同时审阅，输出"必须修改/建议修改/可选优化"三级意见。

这是整个 skill 系列里价值最直接、也最容易被一线教师感受到的工具。 以下是它的完整使用示例：

输入：

以下是一份高一数学"三角函数诱导公式"第一课时的教案，请从多个视角审阅。

---

【导入】（5分钟）
提问：同学们，sin30°等于多少？cos60°呢？有什么关系？
教师口述：sin30°=1/2，cos60°=1/2，所以sin30°=cos60°。
引出课题：今天我们学习诱导公式，看看角的度数之间的关系。

【新知建构】（12分钟）
直接给出公式：sin(90°-α)=cosα，cos(90°-α)=sinα
板书推导过程，教师讲解
举例验证：令α=30°，sin(90°-30°)=sin60°=√3/2，cos30°=√3/2

【例题】（10分钟）
例1：求sin(90°-25°)的值
讲解步骤：先代入公式→再计算
例2：求cos(90°-60°)的值
学生活动：同桌互相出一道类似的题并交换解答

【练习】（10分钟）
给出5道练习题，全是"90°-α"型
学生独立完成，教师巡视
答案全班统一订正

【总结】（3分钟）
教师总结：本节我们学习了两个诱导公式，需要熟练运用。
作业：练习册第28-30页

输出（多视角审阅意见）：

教研员视角：

导入环节用了5分钟，但全程是教师讲授，没有明确的学情诊断依据来支撑"学生已掌握弧度制和象限角"这一判断。建议在导入环节增加一个30秒的前置检测——如"请在单位圆上标出α=30°、60°、90°的位置"，根据举手人数判断是否需要补充弧度制知识。[必须修改]

新知建构直接给出公式，缺少学生自主发现的过程。诱导公式的本质是"角度加减π/2后函数值的变化"，学生如果不知道这个几何直观，直接背公式会很快遗忘。建议增加"单位圆上点的旋转动画"或"用坐标纸描点"的探究活动。[必须修改]

练习题全部是同一题型（90°-α），无法检测学生是否能识别不同的变式方向。[建议修改]

学困生视角：

例题从"90°-α"直接跳到了一般角的诱导公式，中间没有过渡题。我在"90°-α"这里还没完全搞清楚（尤其是"角度相减对应点怎么转"），直接看到一般角的公式会懵。建议在给出一般公式之前，先用几个具体的90°-α的题目让我找到规律，而不是直接告诉我结论。[必须修改]

变式练习5道题全是"已知角求函数值"，没有反向题（已知函数值求角的范围，或者已知函数值判断角的可能取值）。这类题我经常出错，但练习里完全没有涉及。[建议修改]

学优生视角：

整节课的挑战性不够。诱导公式还有 sin(180°-α)=sinα、cos(180°+α)=-cosα 等多组，全部只讲了一组就进入练习，深度没有拉开。建议在学生完成基本练习后，增加一道综合题：sin(90°+30°)怎么求？引导学生发现还有更多的诱导公式需要探究。[建议修改]

例题讲解之后有"同桌互出题"的活动，这个设计很好，但时间只给2分钟就进入统一订正，实际上学生还没有充分讨论。建议延长至4-5分钟，并增加"谁的题最有挑战性"的展示环节。[可选优化]

提示词模板（lesson-reviewer 完整版）：

请扮演"教案预审专家"，从以下四个视角对提交的教案进行审阅。

- 检查教学目标是否清晰、可观测
- 检查学情依据是否充分（是否有前置知识诊断）

**家长视角**（可选）：

- **可选优化**：锦上添花的改进空间

Skill 3：test-analyser（试卷与学情分析）

一句话说清楚： 输入一份试卷和班级成绩数据，输出错误分类统计、各知识点掌握率、可用于下一节课的补偿练习建议。

输入：

试卷：高一数学单元测试（三角函数）
班级均分：72.3
年级均分：68.5
得分率低于60%的题目：第5题（10分）、第11题（12分）、第13题（10分）
第5题错误样本（典型）：多数学生写了sin(90°-α)=cosα，但符号判断错误
第11题错误样本（典型）：学生不知道什么时候用sin(π/2+α)，什么时候用sin(π-α)

输出（示例）：

【整体评估】
班级均分高于年级均分3.8分，表现良好。但标准差偏大（14.2），说明班级内部分化严重。

【核心问题定位】
| 错误类型 | 涉及题目 | 错误率 | 主要认知根源 |
|---|---|---|---|
| 符号判断错误 | 第5题 | 62% | 对"90°+α"和"90°-α"在单位圆上的位置变化方向混淆 |
| 公式选择错误 | 第11题 | 54% | 不理解诱导公式的"奇变偶不变"口诀背后的几何含义 |
| 综合应用缺失 | 第13题 | 71% | 无法将多个诱导公式组合使用 |

【补偿练习建议】
针对第5题（符号判断）：
设计"纠错型"练习：给出3道符号错误的解题过程，让学生指出错误并改正。
例："sin(90°+30°)=sin60°=√3/2，请指出错误并给出正确答案"
针对第11题（公式选择）：
设计"分类选择"练习：给出10道题，每题包含两个选项（分别对应不同公式），学生选择正确公式并说明理由。

【下一节课教学调整建议】
建议在下一节课开始前增加5分钟"诱导公式家族图谱"整理，用结构化板书帮学生建立完整认知框架，而非零散的公式记忆。

提示词模板：

请帮我分析一份试卷的班级学情。

## 选填信息

Skill 4：study-diagnoser（学情前置检测）

一句话说清楚： 输入下一个单元的学习目标，输出一份可用于课堂初始的诊断题清单和判断标准，告诉你学生进教室前已经知道了什么、还不知道什么。

这是衔接两篇文章的关键 skill——行为层用它来采集数据，思维层靠这些数据做判断。

输入：

学科：高中数学
年级：高一
下一单元：平面向量（加减法与数乘运算）
学生即将学习的内容：
- 向量的定义与表示
- 向量加法的三角形法则与平行四边形法则
- 向量数乘运算的几何意义

班级学情背景：刚学完三角函数，运算能力中等，空间想象能力个体差异大。

输出（示例）：

【前置知识检测清单】

目标：判断学生是否具备学习平面向量所需的先备知识

| 检测题 | 对应的先备知识 | 通过标准 | 不通过时的补救建议 |
|---|---|---|---|
| 题1：画一个直角坐标系，标出点A(3,2)和点B(5,4)，向量AB怎么表示？ | 有向线段的概念 | 80%学生能正确表示 | 补充有向线段定义，用物理位移做类比 |
| 题2：计算 3×(2,1) 和 (2,1)×3，结果相同吗？ | 标量乘法基本运算 | 75%学生能正确计算 | 用温度变化类比标量乘法 |
| 题3：在坐标系中画一条从(0,0)到(3,4)的箭头，再画一条从(3,4)到(6,8)的箭头，两条箭头方向相同吗？长度有什么关系？ | 方向与比例直觉 | 60%学生能直觉判断 | 这是本单元的核心障碍点，建议在引入环节重点处理 |

【关键发现】
本班学生在"向量方向"上的直觉较强（题3通过率高于预期），但在"向量坐标表示"上掌握率偏低（题1仅约70%）。建议：引入环节可以从方向入手（题3），再过渡到坐标表示（题1），最后抽象为数乘运算。

【检测时间估算】
题1-3共需约5-7分钟。建议嵌入课堂前5分钟，不单独占用一节课。

① 检测目标：这份检测要判断学生是否具备哪些先备知识
② 检测题清单（3-5道）：每道题注明对应的先备知识、设计理由、通过标准
③ 检测结果解读指南：不同通过率对应什么教学决策（全员通过/部分需要补救/大面积需要重新铺垫）
④ 补救活动建议：针对未通过率高于30%的知识点，给出补救活动的具体形式
⑤ 本检测的局限性说明

## 判断标准
检测题应满足：
- 每道题对应一个明确的先备知识
- 可在5分钟内完成（课堂嵌入可行性）
- 有明确的判断标准（不是开放性问题）

四、怎么判断一个教学经验值不值得做成 skill？

上面四个 skill 是直接可以照做的。但如果你想让教研组自己制作更多 skill，还需要一个判断标准——什么值得做，什么不值得？

nuwa-skill 有一套"三重验证法"，在教学场景下同样适用。一个教学经验要被提炼成 skill，必须同时满足以下三个条件：

条件一：跨情境复现

这个策略在两个以上不同单元被验证过有效？

"例题讲完先让学生出题"这个策略，在三角函数、几何证明、概率统计三个不同单元都有效——这是跨情境复现，是一个真正的教学心智模型。如果只在某道特定题上管用，降级为"具体操作建议"，不值得单独做成 skill。

条件二：有预测力

这个策略能不能预测面对一个新单元时该怎么做？

"核心约束条件识别"这个模型，不仅能解释为什么学生在已学单元上犯错，还能预测他们在下一个新单元哪里会卡住——这就有预测力。如果一个策略只能解释过去发生的事，对未来的教学设计没有指导价值，它就不是心智模型。

条件三：有排他性

不是所有有经验的老师都会这么做，才值得显性化。

"课前要备课"不是心智模型——所有老师都这么做，做成 skill 没有增量价值。"例题和变式之间插入学生出题环节"才是——因为大多数老师讲完例题直接给变式，这个差异才是值得显性记录的教学判断。

三个都满足 → 值得做成 skill，值得投入时间提炼认知框架

仅满足一或二 → 降级为"操作建议"，可以直接写入提示词模板，不需要单独提炼

一个都不满足 → 不收录

五、教研组如何协作制作和维护 skill

skill 不是一个人埋头做出来的。一个有生命力的 skill，需要教研组集体参与。

谁来做什么

角色 职责
主笔 负责提炼自己的教学经验，提供最佳样本（最好的教案、最有效的学情分析记录）
参与者 参与集体讨论，提供不同视角（不同班级、不同学情的反馈）
审核者 集体审核 skill 产出的判断逻辑是否与教研组共识一致

署名规则：每个 skill 的文档封面写明"主笔：[姓名] | 参与讨论：[教研组全体]"。署名权归贡献者，学校享有使用权，不得作为考核或评优的唯一依据。

版本迭代机制

skill 是活的文档，不是写完就归档。每学期至少更新一次，更新触发条件：

  • 重大教材/课标变化
  • 学情发生明显变化（如新接了一个班级）
  • 使用中发现判断逻辑有误

更新时保留历史版本，标注日期和修改原因，便于追溯。

skill 的及格线

不是每条经验都值得做成完整 skill。一个 skill 达到"可用"的最低标准：

  • 有明确输入（教研组其他成员知道什么时候该用它）
  • 有可验证的输出（把输出和实际教学效果对比，方向一致）
  • 有清晰边界（什么场景下它会失效，已标注）

六、skill 做完之后，还需要验证

写完一个 skill 不是终点。你需要测试它。 参照 nuwa-skill 的验证方法，执行以下三种测试：

已知测试（Sanity Check）

选一个教研组里已经设计过的单元，让加载了这个 skill 的 AI 重新设计一遍，跟老教师的实际设计对比。方向一致 → skill 有效。偏离严重 → 回去调整判断规则的权重。

边缘测试（Edge Case）

选一个教研组没有人做过的新题型或新课型，让 skill 推断应该怎么做。期望结果：AI 能给出合理的推断，但会标注"这部分基于XX推断，不确定是否适用"。如果 AI 斩钉截铁地给了一个答案，说明 skill 的诚实边界没写好。

风格测试（Voice Check）

让 skill 输出一段 100 字的教学建议，判断：有没有这个教研组的教学理念特征？是不是通用的 AI 味鸡汤？是不是教科书原文的拼凑？

结语

回到开头的问题：老师的日常工作，哪些值得做成 skill？

不是所有工作都值得。最值得做的是那些高频发生、有标准流程、有最佳实践可循、但目前靠个人经验支撑的环节——新授课设计、教案预审、学情分析、学前诊断。

做好这四个，你的教研组就已经有了一套可运行的"教学智囊团"。

年轻老师备课的时候，可以随时切到这个智囊团问一句：张老师会觉得这个导入太长吗？教研员会怎么评价这个例题选择？学困生在这个环节会卡在哪里？

张老师退休了，但她的视角还在。

今日话题：你们学校教研组最费时间的环节是哪个？如果有一个 skill 可以帮忙，你最希望它解决什么问题？
