# daily-viewpoint | 每日观点技能

## 技能概述

- **版本**：v1.0
- **创建日期**：2026-04-04
- **定位**：每日简报与每日洞察之间的专属内容层
- **受众**：老师、朋友等外部读者（可公开发送）
- **触发方式**：陛下标记完毕后自动执行（或陛下手动触发）
- **输入**：今日简报中被标✅触动的篇目列表
- **输出**：每日观点完整内容 → 写入 `/.daily_viewpoint.md`
- **下游衔接**：触动篇同步启动 collision-analysis skill（与洞察生产并行）

---

## 一、格式规范（对外版本·严格对应简报篇目）

**铁律**：每篇严格对应今日简报的一篇，不得合并多篇，不得额外添加其他文献，不得更改篇目内容。

### 头部

```
📋 每日观点 · YYYY-MM-DD（第N期）

━━━━━━━━━━━━━━━━━━
```

### 每篇结构

```
【序号】【中文标题】（序号与简报完全对应）
原标题：English Original Title（原文照录）
来源：期刊名 ｜ 发表年月（年份）

【这篇在说什么】
（来自今日简报的原三段正文，完整保留，一字不改）

【我的看法】
（100-200字，学术性洞察，有观点有态度）
格式：💡加粗[一句判断性的核心洞察，不超过25字]。然后展开。
禁止：空话/"值得关注"/"我认为"/"我觉得"/泛泛而谈无具体指向

【如果只记住一句话】
（来自简报原句，15-30字，有判断，不中性）

━━━━━━━━━━━━━━━━━━
```

### 尾部

```
📮 往期回顾
（最近3期标题+日期，每条一行）
```

## 二、写作规则

**"我的看法"写作要求**：

1. 开头必须有一句**判断性核心洞察**（≤25字），格式："AI洞察：..."
2. 展开部分必须：
   - 说明这个判断的含义和适用范围
   - 联系K-12教育实际场景
   - 有明确的立场和态度，不中性
3. 禁止出现：
   - "值得关注"/"具有重要意义"/"不容忽视"（空洞）
   - "我认为"/"我觉得"（口语化）
   - "AI可以"/"AI能够"开头（功能描述不等于洞察）
   - 泛泛而谈无具体指向的结论

**好例子**：
> AI洞察：工具铺开了不等于素养提升了，研究者应把评估重心从使用率转向学习成效。
> 当前AI教育的评估指标普遍停留在"用没用"层面，忽视了学习者认知发展的真实轨迹。建议课程评估引入素养前置-后测设计，而非仅统计工具打开次数。

**差例子**：
> 我觉得这篇文章很有意义，AI教育值得重视。（空洞+口语化）

## 三、执行流程

### Step 1：严格收集触动篇（铁律：不多不少）
读取今日简报 `/.daily_briefing.md`，**严格按简报篇目提取**：
- 标✅触动的是哪几篇，就处理哪几篇
- 不多不少，不合并，不额外添加其他文献

### Step 2：读取对应原文库文件
对每篇触动篇，读取对应原文库文件，提取：
- 英文原标题（严格对应简报中的那篇）
- 发表年月
- 三段正文（直接引用简报内容，一字不改）

### Step 3：撰写"我的看法"（格式：💡加粗）
对每篇：
1. 💡**加粗**一句判断性核心洞察（≤25字）
2. 展开100-200字的学术性评论，保留陛下原话的口气
3. 确保与K-12教育场景结合

### Step 4：组装完整内容
- 头部：期号+日期
- 每篇：序号+标题+来源+三段正文+💡我的看法+一句话
- 尾部：往期回顾（最近3期）
- 末尾附：「📮 每日观点 | 转发请注明出处」

### Step 5：写入+告知
1. 写入 `/.daily_viewpoint.md`
2. 通知陛下已生成

## 四、与洞察生产的并行关系

| 环节 | 内容 | 受众 |
|------|------|------|
| 每日观点（刚完成） | 触动篇完整内容+我的看法 | 老师/朋友等外部读者 |
| 碰撞分析（并行） | 触动篇→碰撞卡→洞察 | 陛下内部决策参考 |

两者可同时执行，互不依赖。

## 五、衔接规则

| 上游 | 触发条件 | 下游 |
|------|---------|------|
| 每日简报 | 陛下标记完毕 | 生成每日观点 + 启动碰撞分析 |
| 每日观点 | 完成 | 告知陛下可转发 |
| 碰撞分析 | 收到判断 | 生成洞察全文 |

如陛下回复「跳过每日观点」，则只执行碰撞分析，不生成观点。

## 六、简报提炼版（每次生成每日观点后同步输出）

### 定位
基于每日观点全部篇目，提炼为"标题区+速读区"格式，用于朋友圈/公众号导读/群发，制造点击好奇心，30秒让人掌握全貌。

### 格式规范

标题区（第一行）：
- 从全部文章中挑3篇"信息落差感"最强的内容
- 每篇提炼为10～20字的短句，三句之间用"/"连接
- 每个短句必须包含至少一个钩子：具体数字/对比反差/因果悬念/情绪触发词
- 三句主题尽量分散（不集中同一领域）
- 总字数：30～50字

速读区（标题区下方）：
- 每篇对应一条，emoji开头
- emoji须与内容语义相关，整组不重复
- 每条15～25字
- 句式优先：数字+事件型 / 主体+动作+结果型 / 核心判断型 / 对比反差型
- 优先暴露结论或反直觉事实，不描述过程
- 用大众语言，避免学术术语和缩写

### 输出格式（固定，不可更改）

第一行：标题区（短句A/短句B/短句C）

空一行

速读区（每行一条，emoji开头）

### 存放位置
写入 /.daily_viewpoint_summary.md，告知陛下可直接复制使用
