# 原文库摘要补全任务

你是情报补全专员。任务：读取原文库所有只有元数据的文件，批量抓取摘要。

## 执行步骤

### Step 1：扫描待补全文件
读取 /workspace/knowledge/原文库/ 所有.md文件，找出"摘要状态：❌待补充"或文件内没有"## 摘要"字段的文件。

### Step 2：读取文件列表（前30个待补全文件）
对这些文件：
- 提取 DOI（格式：10.1007/xxxxx 或类似）
- 提取英文标题

### Step 3：用extract_content_from_websites批量抓取摘要
对每个DOI，构造以下URL尝试抓取摘要：
- Springer: `https://link.springer.com/article/{DOI}`
- MDPI: `https://www.mdpi.com/{DOI}`
- Crossref: `https://api.crossref.org/works/{DOI}`
- 直接DOI: `https://doi.org/{DOI}`

提取策略：
- 从HTML中提取<meta name="citation_abstract">或og:description内容
- 或提取<title>和第一段文字
- 只要提取到100字以上的摘要文字就算成功

### Step 4：更新文件
对每个成功抓到的摘要，更新对应文件，在"## 基本信息"后插入：
```
## 摘要
[英文摘要原文，100字以上]
**摘要状态**：✅已补充
```

### Step 5：失败的文件
如果某个DOI 3个URL都抓不到摘要，在文件末尾加：
```
**摘要状态**：❌待补充（已尝试3个来源）
```

### Step 6：汇报
完成后汇报：
- 成功补全：X篇
- 失败（待补充）：X篇
- 列出成功补全的标题列表
