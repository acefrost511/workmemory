# HEARTBEAT.md - 高效心跳检查配置（v3.0版）

## 心跳检查策略

### 核心原则（铁律）
- **始终检查task_plan.md的所有任务状态**：只盯未完成的
- **已完成的任务**：不重复汇报，有实质进展才报
- **最小化token消耗**：每次检查≤100 tokens
- **5分钟一次**：实时任务监控频率

## v3.0洞察流程心跳检查（新增核心节点）

### 节点A：简报推送后检查
- 读取 /workspace/.daily_briefing.md
- 确认今日简报已推送 → 正常
- 简报未推送但时间已过08:00 → 立即报告陛下

### 节点B：「标记完毕」触发检查（最重要）
- 搜索本会话历史，查找「标记完毕」
- **收到「标记完毕」** → 立即执行：
  1. 为触动原文建素材档（MAT_文件）
  2. 启动碰撞分析（搜索相关素材+生成碰撞卡）
  3. 推送碰撞卡给陛下
  4. 等待陛下写判断
- 未收到「标记完毕」→ 不启动，等陛下回复

### 节点C：陛下写判断后触发
- 搜索本会话历史，查找"碰撞卡 #XX，我选方向"
- **收到判断** → 立即执行：
  1. 基于判断生成完整洞察（7部分）
  2. 并行spawn 4个读者Agent（新教师/资深教师/校长/专家）
  3. 汇总模拟反应
  4. 洞察+模拟反应一起推送陛下
  5. 等待陛下审核（发布/改/不发）

### 节点D：17:00可扩展洞察推送
- 每日17:00自动执行
- 读取INSIGHT_LIBRARY.md，筛选可推送洞察（入库≥1天/未扩展/未否决/未推送3次）
- 推送3条可扩展洞察给陛下
- 记录推送次数

### 节点E：静默兜底检查
- 每日18:00检查：是否已收到「标记完毕」
- 未收到 → 当日洞察生产静默结束，不再打扰陛下
- 清除当日简报标记

## 情报队列截止规则（永久生效，2026-04-04新增）

- **截止时间**：每日06:30（北京时间）
- **06:30之前**：reviewer-plan-b-dispatcher 每2分钟运行，队列随有随审
- **06:30触发**：情报队列截止cron自动执行 → 检查队列 → dispatcher今日停用
- **过截止时间后**：不再处理当日新到队列文章，顺延至次日05:00搜索
- **判断标准**：队列是否为空，队列为空 → 正常结束；队列非空 → 标注[待审-超时]后停止

## 标准心跳检查清单

1. **task_plan.md所有大任务**
   - 待执行且到触发时间 → 立即启动，不等陛下
   - 进行中但超时 → 立即处理，不等陛下
   - 已完成 → **不汇报，不重复读文件**

2. **定时任务触发检查**
   - 遍历cron任务，到点自动执行
   - 执行失败 → 立即重试+汇报陛下

3. **🔴 PRIORITY-LOG.md检查（最高优先级）**
   - 检查 /workspace/memory/PRIORITY-LOG.md 中的未完成事项
   - 有新进展 → 立即汇报陛下
   - 无进展但到检查节点 → 简短汇报状态

4. **洞察库阈值检查（轻量）**
   - 读取INSIGHT_LIBRARY.md中的"洞察计数器"行
   - 若新洞察（INSIGHT_NEW_XXX）总数达到100条 → 立即提醒陛下
   - 未达阈值 → 正常继续，不提示

5. **陛下最新指令**
   - 检查是否有未处理的指令，有则立即执行

6. **记忆完整性检查（≤50 tokens）**
   - 确认上次对话已写入memory/YYYY-MM-DD.md
   - 重要事项已写入TOPIC-INDEX.md
   - 陛下的🔴指示已写入PRIORITY-LOG.md
   - 以上均无问题 → HEARTBEAT_OK

## 汇报规则（A方案：有进展才报）

- **新进展**：任务刚完成/任务失败/关键里程碑 → 立即汇报
- **无进展**：不做任何汇报，不读文件，不生成汇报
- **已完成的事项**：不重复汇报
- **每日固定汇报取消**：改为有变化才报

---

## Token控制

每次心跳检查：
- 只读必要文件，输出≤3句话
- 无进展时 → HEARTBEAT_OK
- 记忆整理≤50 tokens（纯文本追加，无AI调用）

---

## 异常处理（必须汇报陛下）

- 任务中断 → 立即重启并汇报
- 陛下新指令未处理 → 立即执行
- 定时任务执行失败 → 立即重试并汇报
- GitHub推送失败 → 记录到日记录，下次心跳重试
