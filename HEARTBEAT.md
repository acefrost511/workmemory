# HEARTBEAT.md - 高效心跳检查配置（陛下定制版）

## 心跳检查策略

### 核心原则（铁律）
- **始终检查task_plan.md的所有任务状态**：只盯未完成的
- **已完成的任务**：不重复汇报，有实质进展才报
- **最小化token消耗**：每次检查≤100 tokens
- **5分钟一次**：实时任务监控频率

---

### 检查清单（每次心跳必须执行）

1. **task_plan.md所有大任务**
   - 待执行且到触发时间 → 立即启动，不等陛下
   - 进行中但超时 → 立即处理，不等陛下
   - 已完成 → **不汇报，不重复读文件**

2. **定时任务触发检查**
   - 遍历cron任务，到点自动执行
   - 执行失败 → 立即重试 + 汇报陛下

3. **陛下最新指令**
   - 检查是否有未处理的指令，有则立即执行

4. **记忆完整性检查（轻量，不超50 tokens）**
   - 确认上次对话内容已写入topics文件（臣已每轮实时写入，这里做兜底检查）
   - 如有遗漏（异常中断等），立即补充到对应topics文件
   - 确认MEMORY.md索引已同步更新
   - 以上均无问题时 → HEARTBEAT_OK

---

### 汇报规则（陛下定制版 A方案）

**有进展才报，无进展不说，重复内容绝对不报：**

- **新进展**：任务刚完成 / 任务失败 / 关键里程碑到达 → 立即向陛下汇报
- **无进展**：不做任何汇报，不读文件，不生成汇报
- **已完成的事项**：不重复汇报，陛下不主动问就不提
- **每日固定汇报取消**：改为有变化才报

---

### 汇报格式要求

有进展时，汇报必须包含：
- 哪个任务有新进展
- 结果是什么
- 下一步是什么

无进展时：**完全不说，HEARTBEAT_OK**

---

### Token控制

每次心跳检查：
- 只读task_plan.md（不读测试报告、目录列表）
- 输出≤3句话
- 无进展时 → HEARTBEAT_OK
- 记忆整理操作：≤50 tokens额外消耗（纯文本追加，无AI调用）

---

### 异常处理（必须汇报陛下）

- 任务中断 → 立即重启并汇报
- 陛下新指令未处理 → 立即执行
- 定时任务执行失败 → 立即重试并汇报
- GitHub推送失败 → 记录到日记录，下次心跳重试

---

### 记忆整理优先级规则

**永久保留（不压缩）：**
- 方法论/框架/规则变更
- 陛下方重要观点/洞察
- 创作产出物路径
- 重要决策结论
- 任务完成状态变更

**压缩处理（保留精华）：**
- 讨论过程 → 只保留结论
- 待定事项 → 只保留未解决状态
- 临时性确认 → 不保留

**不记录：**
- 日常问候/确认
- 重复确认已知信息
- 工具执行结果（路径已存档的除外）
