# HEARTBEAT.md - 高效心跳检查配置

## 心跳检查策略（优化版）

### 核心原则
- **只检查进行中的任务**：不扫描全部历史
- **只检查最新指令**：关注陛下最新的任务要求
- **最小化token消耗**：每次检查≤100 tokens
- **5分钟一次**：实时任务监控频率

### 检查清单（极简版）

每次心跳只检查：
1. **当前进行中的任务** - task_plan.md中的未完成项
2. **陛下最新的3条指令** - 避免遗漏
3. **任务中断风险** - 长时间无进展的任务

### 任务状态追踪

在task_plan.md中标记：
- `[进行中]` - 正在执行的任务
- `[待处理]` - 陛下最新指令
- `[已完成]` - 已完成任务

### 异常处理

如果发现：
- 任务超过15分钟无进展 → 自动继续推进
- 任务中断 → 立即重启并汇报
- 陛下新指令未处理 → 立即执行

### Token控制

每次心跳检查：
- 只读关键文件（task_plan.md、最新指令）
- 不读取完整历史
- 输出简洁（≤3句话）
